# 품질 분석 파이프라인

## 1. 이 파이프라인의 목적

코드 품질 분석 도구(SonarQube)는 소스 코드에서 버그, 보안 취약점, 유지보수가 어려운 코드를 자동으로 찾아냅니다. 그러나 분석 결과를 읽고, 각 문제의 원인을 파악하고, 이슈 관리 시스템에 수정 요청을 등록하는 과정은 여전히 사람이 수행해야 합니다. 이슈가 수십 건이면 이 과정만으로 반나절이 소요되기도 합니다.

품질 분석 파이프라인은 이 전체 과정을 자동화합니다. 코드 분석 도구가 발견한 문제를 AI가 읽고, 문제의 원인과 수정 방안을 프로젝트 맥락에 맞게 작성하여, 이슈 관리 시스템에 자동으로 등록합니다.

---

## 2. 동작 흐름

이 파이프라인은 세 단계를 거쳐 동작합니다.

### 1단계: 문제 수집

시스템이 코드 분석 도구에 접속하여, 지정된 프로젝트에서 발견된 코드 문제 목록을 가져옵니다. 이때 각 문제에 해당하는 실제 코드 라인도 함께 추출되어, AI가 코드를 직접 보면서 진단할 수 있도록 준비합니다.

사용자는 어떤 수준의 문제를 처리할 것인지 선택할 수 있습니다. 운영 장애를 유발할 수 있는 치명적 결함만 추출할 수도 있고, 코딩 스타일 수준의 경미한 문제까지 포함시킬 수도 있습니다.

### 2단계: AI 진단

수집된 각 문제를 AI가 분석합니다. AI는 코드 분석 도구가 제공하는 규칙 설명을 단순히 번역하는 것이 아니라, 해당 코드가 왜 문제인지, 방치하면 어떤 위험이 있는지, 구체적으로 어떻게 수정하면 되는지를 자연어로 작성합니다.

지식 관리 파이프라인으로 프로젝트의 문서와 코드를 사전에 학습시켜 둔 경우, AI는 프로젝트의 설계 의도와 코드 구조를 이해한 상태에서 진단하므로, "이 메서드를 두 개로 분리하라" 수준의 일반적인 조언이 아닌, "AuthService의 validateToken()과 PermissionChecker의 check()로 분리하면 된다"와 같이 프로젝트에 맞는 구체적인 수정 방안을 제안할 수 있습니다.

### 3단계: 이슈 자동 등록

AI의 진단 결과가 이슈 관리 시스템(GitLab)에 자동으로 등록됩니다. 각 이슈에는 문제의 원인 분석, 영향 범위, 수정 방안, 해당 코드 위치가 포함되어 있어, 담당 개발자가 이슈를 열어보면 별도의 분석 없이 바로 수정 작업에 착수할 수 있습니다.

동일한 문제에 대한 이슈가 중복으로 등록되지 않도록 자동 방지 기능이 내장되어 있으므로, 파이프라인을 반복 실행해도 안전합니다.

---

## 3. 사용 방법

1. Jenkins에서 "DSCORE-Quality-Issue-Workflow" 작업을 선택합니다.
2. 분석 대상 프로젝트와 처리할 문제의 심각도 범위를 지정합니다.
3. 실행 버튼을 클릭합니다.
4. 실행이 완료되면 GitLab의 이슈 목록에서 AI가 작성한 이슈를 확인합니다.

처음 실행할 때는 치명적 수준의 문제만 소량(5건 정도)으로 시작하여 AI의 진단 품질을 확인한 후, 점차 범위를 넓혀 가는 것을 권장합니다.

정기 실행을 설정하면 매일 또는 매주 자동으로 분석을 수행하여, 새로 발생한 코드 문제가 자동으로 이슈로 등록됩니다.

---

## 4. 이 파이프라인을 통해 가능해지는 것

이 파이프라인이 자동으로 등록하는 이슈에는 다음과 같은 내용이 포함됩니다.

- 문제의 근본 원인에 대한 AI의 분석 결과
- 해당 문제를 방치할 경우 발생할 수 있는 구체적인 위험
- 프로젝트 맥락을 반영한 수정 방안 (코드 수정 예시 포함)
- 문제가 발생한 파일명과 코드 라인
- 코드 분석 도구에서 해당 문제를 직접 확인할 수 있는 링크

이를 통해 다음과 같은 업무 흐름이 가능해집니다.

스프린트마다 이 파이프라인을 실행하면, AI가 코드에서 발견된 문제를 자동으로 정리하여 이슈 관리 시스템에 등록합니다. 개발팀은 이슈 목록을 확인하고 우선순위에 따라 수정 작업을 진행하면 됩니다. 코드 분석 결과를 수동으로 읽고, 이슈를 작성하고, 중복을 확인하는 반복 작업이 제거됩니다.

보안 감사가 예정되어 있다면, 치명적 보안 취약점만 추출하여 사전에 대응할 수 있습니다. 신규 프로젝트의 코드 품질 수준을 파악하고 싶을 때는, 전체 이슈를 한 번에 추출하여 현황을 객관적으로 확인할 수 있습니다.

---

## 5. 참고 사항

- AI의 수정 제안은 참고 자료이며, 최종 수정 판단은 담당 개발자가 프로젝트 상황에 맞게 결정하십시오.
- 지식 관리 파이프라인으로 프로젝트를 사전에 학습시키면 AI의 진단 품질이 크게 향상됩니다.
- 모든 데이터 처리는 사내 로컬 서버에서 이루어지며, 코드와 분석 결과가 외부로 전송되지 않습니다.
