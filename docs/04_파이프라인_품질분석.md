# 품질 분석 파이프라인

## 1. 파이프라인의 목적

코드 정적 분석 도구는 소스 코드에 내재된 버그, 보안 취약점, 유지보수성을 저해하는 요소를 효과적으로 탐지합니다. 
그러나, 분석 도구가 생성한 결과를 검토하고, 각 문제의 원인을 파악하여 이슈 관리 시스템에 수정 과업으로 등록하는 후속 과정은 상당 부분 수동 작업에 의존하고 있습니다. 
다수의 이슈를 처리할 경우, 이 과정은 상당한 시간을 소모하여 개발자의 핵심 업무 집중을 방해하는 병목이 됩니다.

품질 분석 파이프라인은 정적 분석 결과의 해석부터 이슈 관리 시스템에 등록하기까지의 전 과정을 자동화하는 것을 목적으로 합니다. 
AI가 분석 결과를 해석하여 개발자가 즉시 조치할 수 있는 형태로 가공 및 등록함으로써, 코드 품질 관리의 피드백 루프를 가속화하고 수작업을 최소화합니다.

---

## 2. 동작 흐름

```
[정적 분석 도구 (SonarQube)]
- 버그, 취약점, 코드 스멜 목록
      │
      ▼ (1. 문제 수집)
[품질 분석 파이프라인]
- AI가 문제의 원인/위험/수정방안 분석
- (사전 학습된 프로젝트 지식 활용)
      │
      ▼ (2. AI 진단 & 3. 이슈 등록)
[이슈 관리 시스템 (GitLab)]
- 담당자, 우선순위 지정 가능한 이슈 자동 생성
- 코드 위치, 수정 가이드 포함
```

이 파이프라인은 아래 3단계의 절차로 동작합니다.

### 단계 1: 정적 분석 결과 수집

파이프라인이 실행되면, 먼저 코드 분석 도구(SonarQube)의 API를 통해 지정된 프로젝트에서 탐지된 문제 목록을 수집합니다. 
이 과정에서 각 문제에 해당하는 소스 코드 라인을 함께 추출하여, AI가 진단 시 실제 코드를 직접 참조할 수 있도록 준비합니다. 
사용자는 파라미터를 통해 분석 대상이 될 이슈의 심각도(예: Blocker, Critical) 범위를 지정할 수 있습니다.

### 단계 2: AI 기반 자동 진단

수집된 각 코드 이슈에 대해 AI가 심층 분석을 수행합니다. 
AI는 코드 분석 도구의 규칙 설명과 함께, 해당 문제가 발생하는 근본적인 원인과 잠재적 위험, 그리고 프로젝트의 컨텍스트를 반영한 구체적인 수정 방안을 분석하여 제시합니다.

특히, 사전에 지식 관리 파이프라인을 통해 프로젝트의 설계 문서나 코드 컨벤션을 AI에게 학습시킨 경우, AI는 프로젝트의 고유한 아키텍처와 설계 의도를 이해한 상태에서 진단하므로, 일반적인 제안을 넘어 해당 프로젝트에 최적화된 수정 방안을 제안할 수 있습니다.

### 단계 3: 이슈 관리 시스템에 자동 등록

AI의 진단 결과는 지정된 이슈 관리 시스템(GitLab)에 표준화된 형식의 이슈로 자동 등록됩니다. 
생성되는 각 이슈에는 다음과 같은 상세 정보가 포함됩니다.

- 문제의 근본 원인 분석
- 방치 시 발생 가능한 잠재적 위험
- 프로젝트 맥락을 고려한 코드 수정 제안
- 문제가 발생한 소스 파일 및 코드 라인 정보
- 원본 정적 분석 결과를 확인할 수 있는 링크

또한, 동일한 문제에 대해 중복 이슈가 생성되는 것을 방지하는 기능이 내장되어 있어, 파이프라인을 주기적으로 반복 실행하여도 안전합니다.

---

## 3. 사용 방법

품질 분석은 **분석 실행**과 **결과 처리**의 두 단계로 진행됩니다.

### 단계 1: 코드 정적 분석 실행

- **목적**: SonarQube를 사용하여 소스 코드의 버그, 취약점, 코드 스멜을 탐지합니다.
- **실행 방법**:
    1. Jenkins에서 **`DSCORE-TTC 코드 정적분석`** 작업을 선택합니다.
    2. 파라미터로 분석할 소스 코드의 경로를 지정하고 실행합니다.
    3. 실행이 완료되면 SonarQube 대시보드에서 분석 결과를 확인할 수 있습니다.

### 단계 2: 분석 결과 AI 진단 및 이슈 등록

- **목적**: 정적 분석 결과를 AI가 진단하고, 조치가 필요한 항목을 GitLab 이슈로 자동 등록합니다.
- **실행 방법**:
    1. Jenkins에서 **`DSCORE-TTC 코드 정적분석 결과분석 및 이슈등록`** 작업을 선택합니다.
    2. 파라미터로 분석 대상 프로젝트와 처리할 이슈의 심각도 범위를 지정합니다.
    3. 작업을 실행합니다.
    4. 실행 완료 후, GitLab 프로젝트의 이슈 목록에서 AI가 등록한 이슈들을 확인합니다.

**권장 사항**
최초 도입 시에는 심각도가 높은 소수의 이슈(예: 5건)를 대상으로 `결과분석 및 이슈등록` 파이프라인을 실행하여 AI의 진단 품질을 검토한 후, 점진적으로 분석 범위를 확대할 것을 권장합니다. 
주기적인 실행을 설정하면, 신규로 발생하는 코드 품질 이슈를 자동으로 추적하고 관리할 수 있습니다.

## 4. 주요 기대 효과 및 활용 시나리오

본 파이프라인을 통해 정적 분석 결과 처리 과정을 자동화함으로써 다음과 같은 효과를 기대할 수 있습니다.

- **개발 생산성 향상:** 개발팀은 정적 분석 결과를 수동으로 검토하고 이슈를 작성하는 반복적인 업무에서 벗어나, 생성된 이슈를 바탕으로 즉시 수정 작업에 착수할 수 있습니다.
- **코드 품질 관리 강화:** 주기적인 자동 분석 및 리포팅을 통해, 기술 부채가 축적되기 전에 문제를 조기에 발견하고 체계적으로 관리할 수 있는 프로세스를 구축합니다.
- **보안 감사 및 품질 현황 파악:** 보안 감사 기간에는 보안 취약점(Vulnerability) 유형의 이슈만 필터링하여 사전 대응할 수 있으며, 신규 프로젝트의 초기 품질 수준을 파악하기 위해 전체 이슈를 추출하여 현황을 객관적으로 분석하는 데 활용할 수 있습니다.

---

## 5. 참고 사항

- AI가 제안하는 수정 방안은 권고 사항이며, 실제 코드에 적용할지 여부에 대한 최종 판단과 책임은 담당 개발자에게 있습니다.
- 지식 관리 파이프라인을 통해 프로젝트 관련 지식을 사전에 학습시키면 AI의 진단 정확성과 제안의 구체성이 크게 향상됩니다.
- 모든 데이터 처리는 사내 로컬 서버에서 이루어지므로, 소스 코드와 분석 결과가 외부로 전송되지 않습니다.
