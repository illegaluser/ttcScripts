# 지식 관리 파이프라인

## 1. 파이프라인의 목적

소프트웨어 프로젝트가 진행됨에 따라, 관련 지식 자산(기획서, 설계서, 소스 코드, 기술 자료 등)은 다양한 시스템에 분산되어 저장되는 경향이 있습니다. 
이러한 지식의 파편화는 정보 접근성을 저해하고, 특정 정보나 담당자를 찾는 데 불필요한 시간을 소모시켜 팀 전체의 생산성을 저하시키는 원인이 됩니다.

지식 관리 파이프라인은 이렇게 분산된 프로젝트 지식 자산을 AI가 학습 가능한 형태로 변환하여, 중앙화된 단일 지식 베이스를 구축하는 것을 목적으로 합니다. 
학습이 완료된 AI는 프로젝트 전반에 걸친 질문에 대해 여러 지식 소스를 동시에 참조하여, 컨텍스트에 맞는 종합적인 답변을 제공할 수 있습니다.

---

## 2. 동작 흐름

```
[입력 소스]
- 문서 (PDF, Word)
- 소스코드 (Git)
- 웹 콘텐츠 (Wiki)
      │
      ▼
[지식 처리 파이프라인]
1. 텍스트 추출/정제
2. 질문/답변 생성 (선택)
3. 청킹 & 임베딩
      │
      ▼
[AI 지식 베이스]
(벡터 DB)
      │
      ▼
[AI 에이전트]
- 프로젝트 특화 답변 제공
```

본 파이프라인은 다양한 유형의 지식 소스를 처리하기 위한 3개의 학습 경로를 제공합니다.

### 경로 1: 문서 학습 (Jenkins Job #1, #2)

프로젝트 관련 문서를 지정된 공유 폴더에 배치하고 파이프라인을 실행하면, 시스템이 문서 종류(PDF, Word, Excel, PPT, TXT 등)를 자동으로 식별하여 텍스트 콘텐츠를 추출합니다. 
이 과정에서 사용자의 별도 변환 작업은 필요하지 않으며, PDF 내 표나 차트 등의 객체도 텍스트로 변환됩니다. 추출된 내용은 AI가 검색할 수 있는 지식 베이스에 저장됩니다.

- **Job #1 (Full-text 기반):** 문서 원문을 그대로 학습하여, 특정 구절이나 키워드를 정확하게 찾아내는 검색에 최적화된 방식으로 처리합니다.
- **Job #2 (Q&A 기반):** 문서 내용을 기반으로 예상 질문과 답변 쌍을 AI가 미리 생성하여 학습합니다. 이는 "사용 방법"과 같은 실무적이고 의미 중심적인 질문에 더 높은 정확도의 답변을 제공합니다.

두 작업을 모두 실행할 경우, AI의 답변 품질과 정보 접근성을 극대화할 수 있습니다.

### 경로 2: 소스 코드 학습 (Jenkins Job #3)

지정된 소스 저장소(Git Repository)를 대상으로, 프로젝트의 전체 코드 구조와 핵심 로직을 AI에게 학습시킵니다. 
파이프라인 실행 시, 시스템은 코드의 폴더 구조, 주요 설정 파일, 핵심 소스 코드를 분석하여 구조화된 요약 문서를 생성하고, 이를 지식 베이스에 저장합니다. 
이를 통해 코드의 구조적 현황이나 특정 모듈의 위치 등에 대한 질의에 대응할 수 있습니다.

### 경로 3: 웹 콘텐츠 학습 (Jenkins Job #5)

사내 위키, 기술 블로그 등 웹에 게시된 정보를 수집하여 지식 베이스에 추가합니다. Jenkins에 수집 대상 URL을 지정하고 실행하면, 시스템은 해당 페이지와 하위 페이지의 본문 텍스트를 크롤링합니다. 
이 과정에서 메뉴, 광고, 푸터 등 비본질적인 HTML 요소는 자동으로 필터링되며, 정제된 텍스트만 지식으로 저장됩니다.

## 3. 사용 방법

### 문서 학습

1. 학습시킬 문서를 지정된 공유 문서 폴더에 배치합니다.
2. Jenkins에서 `DSCORE-Knowledge-Sync` 작업을 선택하고 실행합니다.
3. (선택) 답변 품질 향상을 위해, `DSCORE-Knowledge-Sync-QA` 작업을 추가로 실행합니다.
4. 실행 완료 후, AI 채팅 화면에서 관련 내용을 질의하여 학습 결과를 확인합니다.

### 소스 코드 학습

1. Jenkins에서 `DSCORE-Code-Knowledge-Sync` 작업을 선택합니다.
2. 파라미터로 분석할 소스 저장소의 주소를 입력하고 실행합니다.
3. 실행 완료 후, AI에게 프로젝트 구조에 관한 내용을 질의하여 학습 결과를 확인합니다.

### 웹 콘텐츠 학습

1. Jenkins에서 `DSCORE-Web-Knowledge-Sync` 작업을 선택합니다.
2. 파라미터로 수집할 웹 페이지의 시작 주소를 입력하고 실행합니다.
3. 실행 완료 후, AI에게 해당 웹 페이지 내용에 관해 질의하여 학습 결과를 확인합니다.

최초 지식 베이스 구축 시에는 모든 파이프라인을 순차적으로 실행하는 것을 권장하며, 이후에는 변경이 발생한 지식 소스에 해당하는 파이프라인만 선택적으로 재실행하여 최신 상태를 유지할 수 있습니다.

## 4. 주요 기대 효과 및 활용 시나리오

본 파이프라인을 통해 구축된 지식 베이스는 다음과 같은 활용 시나리오에서 가치를 제공합니다.

- **프로젝트 컨텍스트에 기반한 질의응답:**
  - "프로젝트의 인증 방식은 무엇입니까?" 와 같은 질문에 대해, AI가 관련 설계서와 실제 코드 구현을 종합하여 답변을 생성합니다.
  - "결제 모듈의 전체적인 데이터 흐름을 설명해 주십시오." 와 같은 복합적인 질문에, 기획서의 프로세스 정의와 코드의 데이터 구조를 함께 분석하여 요약된 정보를 제공합니다.

- **신규 팀원 온보딩 지원:**
  - 신규 입사자가 프로젝트의 기술 스택, 아키텍처, 주요 비즈니스 로직을 파악하기 위해 여러 담당자를 찾아다니는 대신, AI에게 관련 내용을 직접 질의하여 학습 시간을 단축할 수 있습니다.

- **다른 파이프라인과의 연계를 통한 시너지:**
  - 품질 분석 파이프라인(Job #4) 실행 전 본 파이프라인을 통해 프로젝트 지식을 AI에게 사전 학습시키면, AI가 코드의 설계 의도를 이해한 상태에서 정적 분석 결과를 진단하므로, 더 구체적이고 정확도 높은 수정 방안을 제안할 수 있습니다.

---

## 5. 참고 사항

- 모든 데이터 처리 및 AI 학습은 사내 로컬 서버에서만 이루어지며, 문서와 코드가 외부로 전송되지 않아 높은 수준의 보안을 유지합니다.
- 학습된 내용은 해당 AI 애플리케이션을 사용하는 모든 구성원이 접근할 수 있으므로, 지식 베이스에 포함되는 정보의 보안 등급을 사전에 확인해야 합니다.
- 기존 문서가 갱신된 경우, 동일한 학습 파이프라인을 재실행하면 최신 버전의 내용이 지식 베이스에 반영됩니다.
