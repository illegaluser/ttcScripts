# 지식 관리 파이프라인

## 1. 이 파이프라인의 목적

프로젝트를 진행하다 보면 지식은 자연스럽게 흩어집니다. 기획서는 공유 드라이브에, 코드는 소스 저장소에, 기술 참고 자료는 사내 위키에 각각 존재합니다. 누군가 "이 프로젝트의 인증 방식이 뭐야?"라고 물으면, 문서를 뒤지고, 코드를 열어보고, 위키를 검색해야 비로소 답을 얻을 수 있습니다.

지식 관리 파이프라인은 이렇게 흩어져 있는 프로젝트 지식을 AI에게 학습시키는 기능입니다. 한 번 학습이 완료되면, AI는 프로젝트 전반에 걸친 질문에 대해 문서와 코드를 동시에 참조하여 답변할 수 있게 됩니다. 일반적인 AI가 교과서적인 답변만 제공하는 것과 달리, 학습을 마친 AI는 "우리 프로젝트"에 특화된 구체적인 답변을 제공합니다.

---

## 2. 동작 흐름

이 파이프라인은 네 가지 학습 경로를 제공하며, 각각 다른 유형의 지식을 AI에게 전달합니다.

### 문서 학습 (Job #1, Job #2)

사용자가 프로젝트 관련 문서를 지정된 공유 폴더에 넣고 Jenkins에서 실행 버튼을 누르면, 시스템이 해당 문서의 텍스트를 자동으로 추출합니다. PDF 안에 포함된 표나 차트도 AI가 인식하여 텍스트로 변환합니다. 추출된 내용은 AI의 지식 저장소에 적재되며, 이후 AI는 해당 문서의 내용을 근거로 답변할 수 있게 됩니다.

Job #1은 문서의 원문을 그대로 학습시키는 방식입니다. AI가 문서의 특정 구절을 직접 참조하여 답변하므로, "설계서에 뭐라고 적혀 있는가"와 같은 질문에 적합합니다.

Job #2는 동일한 문서를 바탕으로 AI가 예상 질문과 모범 답변을 미리 정리해 두는 방식입니다. "이 기능은 어떻게 사용하면 되는가"와 같은 실무적인 질문에 더 정확한 답변을 제공합니다. Job #1과 Job #2를 모두 실행하면 AI의 답변 품질이 가장 높아집니다.

지원하는 문서 형식은 PDF, Word, Excel, PowerPoint, 텍스트 파일입니다. 사용자가 별도의 형식 변환을 할 필요는 없으며, 시스템이 각 형식에 맞는 텍스트 추출을 자동으로 처리합니다.

### 소스 코드 학습 (Job #3)

소스 저장소에 등록된 프로젝트의 코드 구조를 AI에게 학습시킵니다. Jenkins에서 저장소 주소를 지정하고 실행하면, 시스템이 해당 프로젝트의 폴더 구조, 주요 설정 파일, 핵심 소스 코드를 분석하여 하나의 요약 문서를 생성합니다. 이 문서가 AI의 지식 저장소에 적재되면, AI는 "이 프로젝트는 어떤 구조로 되어 있는가", "인증 모듈은 어디에 있는가"와 같은 코드 구조 관련 질문에 답변할 수 있게 됩니다.

### 웹 콘텐츠 학습 (Job #5)

사내 위키, 기술 블로그, 외부 참고 문서 등 웹에 게시된 정보를 AI에게 학습시킵니다. Jenkins에서 수집할 웹 페이지의 주소를 지정하고 실행하면, 시스템이 해당 페이지와 연결된 하위 페이지의 본문 텍스트를 자동으로 수집합니다. 메뉴, 광고, 꼬리말 등 불필요한 요소는 자동으로 제거되며, 정제된 본문만 AI의 지식 저장소에 적재됩니다.

---

## 3. 사용 방법

### 문서 학습

1. 학습시킬 문서를 프로젝트의 공유 문서 폴더에 넣습니다.
2. Jenkins에서 "DSCORE-Knowledge-Sync" 작업을 선택하고 실행합니다.
3. 실행이 완료되면 AI 채팅 화면에서 해당 문서에 대한 질문을 던져 학습 결과를 확인합니다.
4. (선택) "DSCORE-Knowledge-Sync-QA" 작업을 추가로 실행하면, 질문-답변 형태의 학습이 보강되어 AI의 답변 정확도가 향상됩니다.

### 소스 코드 학습

1. Jenkins에서 "DSCORE-Code-Knowledge-Sync" 작업을 선택합니다.
2. 분석할 소스 저장소의 주소를 입력하고 실행합니다.
3. 실행이 완료되면 AI에게 프로젝트 구조에 관한 질문을 던져 학습 결과를 확인합니다.

### 웹 콘텐츠 학습

1. Jenkins에서 "DSCORE-Web-Knowledge-Sync" 작업을 선택합니다.
2. 수집할 웹 페이지의 주소를 입력하고 실행합니다.
3. 실행이 완료되면 AI에게 해당 웹 페이지의 내용에 대한 질문을 던져 학습 결과를 확인합니다.

처음 지식 베이스를 구축할 때는 문서 학습(Job #1, #2) → 코드 학습(Job #3) → 웹 학습(Job #5) 순서로 실행하는 것을 권장합니다. 이후에는 변경이 발생한 항목에 해당하는 작업만 재실행하면 됩니다.

## 4. 이 파이프라인을 통해 가능해지는 것

학습이 완료된 AI에게는 다음과 같은 질문이 가능합니다.

- "이 프로젝트의 인증 방식은 무엇인가요?" — AI가 설계서의 관련 항목과 실제 코드의 구현 위치를 동시에 참조하여 답변합니다.
- "결제 모듈의 전체 흐름을 설명해 주세요." — 기획서에 기재된 업무 흐름과 코드 구조를 종합하여 요약합니다.
- "사용자 권한 체계가 어떻게 되어 있나요?" — 요구사항 정의서의 권한 목록과 소스 코드의 권한 검사 로직을 함께 설명합니다.
- "최근에 변경된 API 스펙이 있나요?" — 학습된 문서 중 관련 내용을 찾아 답변합니다.

하나의 질문에 문서, 코드, 위키의 정보를 통합하여 답변할 수 있다는 점이 이 파이프라인의 핵심 가치입니다. 프로젝트에 신규 인력이 합류했을 때, 이 AI를 통해 프로젝트의 구조와 맥락을 빠르게 파악할 수 있으며, 기존 구성원도 흩어진 정보를 일일이 찾는 대신 AI에게 질문하여 시간을 절약할 수 있습니다.

또한, 이 파이프라인으로 프로젝트 지식을 학습시켜 둔 상태에서 품질 분석 파이프라인(Job #4)을 실행하면, AI가 프로젝트의 코드와 설계 의도를 이해한 상태로 코드 문제를 진단하므로 진단의 구체성과 정확도가 크게 향상됩니다.

---

## 5. 참고 사항

- 모든 데이터 처리는 사내 로컬 서버에서 이루어지며, 학습시킨 문서와 코드가 외부로 전송되지 않습니다.
- 학습된 내용은 해당 AI 앱을 사용하는 모든 구성원이 조회할 수 있으므로, 사내 기밀 문서를 학습시키기 전에 보안 정책을 확인하십시오.
- 문서가 갱신된 경우 동일한 작업을 다시 실행하면 최신 내용이 반영됩니다.
