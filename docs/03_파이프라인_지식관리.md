# 지식 관리 파이프라인 (Job #1~#3, #5)

AI가 우리 프로젝트를 이해하도록 만드는 기능입니다.

---

## 이런 경험, 한 번쯤 있지 않으셨나요?

신입 사원이 들어왔습니다. 프로젝트에 대해 물어보고 싶은 게 산더미인데, 정작 문서가 어디에 있는지 아무도 정확히 모릅니다. 누군가가 "그건 위키에 있을걸?" 하면, 다른 사람은 "아니, 기획서에 있었는데..." 하고, 결국 선임 개발자를 붙잡고 물어보는 것으로 끝납니다.

혹은 이런 상황은 어떠신가요?

- "이 기능이 어떻게 구현되어 있어?" 라고 물으면 돌아오는 대답은 "코드 보면 돼"
- 사내 위키에 정보가 분명히 있는데, 어디에 있는지 찾는 데만 30분
- 기획서는 PDF로, 코드는 GitLab에, 기술 문서는 위키에 흩어져 있어서 전체 그림을 그리기 어려움
- AI에게 프로젝트에 대해 질문하면 인터넷에서 긁어온 일반론만 답변

정보가 없는 것이 아닙니다. 정보는 있는데 흩어져 있고, 찾기 어렵고, 사람의 머릿속에만 존재하는 것이 문제입니다.

지식 관리 파이프라인은 바로 이 문제를 해결합니다.

---

## 이 기능이 해결하는 문제

핵심은 간단합니다. 우리 프로젝트의 문서, 코드, 위키 내용을 AI에게 학습시켜서, AI가 "우리 프로젝트 전문가"가 되도록 만드는 것입니다.

학습 전과 후의 차이를 비교해 보겠습니다.

### 학습 전 -- 인터넷 검색 수준의 답변

질문: "우리 프로젝트 인증 방식이 뭐야?"

AI: "일반적으로 웹 애플리케이션에서 사용하는 인증 방식에는 토큰 기반, 세션 기반, 소셜 로그인 등이 있습니다..."

이것은 우리 프로젝트에 대한 답변이 아닙니다. 누구에게나 동일하게 나오는 교과서적인 답변입니다.

### 학습 후 -- 우리 프로젝트 기반의 답변

질문: "우리 프로젝트 인증 방식이 뭐야?"

AI: "현재 프로젝트에서는 JwtTokenProvider.java에서 토큰 기반 인증을 사용하고 있으며, 토큰 만료 시간은 설정 파일에서 3600초로 지정되어 있습니다. 기획서 3.2절 '보안 요구사항'에 따르면, 소셜 로그인도 병행하도록 설계되어 있습니다."

코드와 문서를 동시에 참조하여, 우리 프로젝트에 맞는 구체적인 답변을 제공합니다. 일반적인 AI와 "우리 프로젝트를 아는 AI"의 차이가 바로 여기에 있습니다.

---

## 무엇을 학습시킬 수 있나요?

| 학습 대상 | 예시 | 담당 작업 |
| --- | --- | --- |
| 프로젝트 문서 | 기획서, 설계서, 매뉴얼, 보고서 (PDF, Word, Excel, PowerPoint, 텍스트) | Job #1, #2 |
| 소스 코드 구조 | GitLab에 있는 프로젝트의 폴더 구조와 핵심 설정 파일 | Job #3 |
| 웹 페이지 | 사내 위키, 기술 블로그, 외부 참고 문서 | Job #5 |

각 작업은 독립적으로 실행할 수 있으며, 필요에 따라 조합하여 사용하면 AI의 답변 품질이 더욱 높아집니다. 예를 들어, 프로젝트 문서(Job #1)와 소스 코드(Job #3)를 모두 학습시키면, AI가 "기획서에는 이렇게 정의되어 있고, 실제 코드에서는 이렇게 구현되어 있다"는 식의 통합적인 답변을 제공할 수 있습니다.

---

## 사용 방법

### Job #1: 프로젝트 문서 학습시키기

프로젝트에서 사용하는 기획서, 설계서, 매뉴얼 등 문서 파일을 AI에게 읽히는 기능입니다. 문서를 넣고 버튼 하나만 누르면 됩니다.

**준비할 것:** 학습시키고 싶은 문서 파일들 (PDF, Word, Excel, PowerPoint, 텍스트)

**절차:**

**1단계. 학습시킬 문서를 지정된 공유 폴더에 넣습니다.**

Finder에서 아래 경로로 이동하여 문서 파일을 넣어 주십시오.

- 폴더 위치: (프로젝트 루트)/data/knowledges/docs/org/
- 지원 형식: PDF, Word(.docx), Excel(.xlsx, .xls), PowerPoint(.pptx), 텍스트(.txt)

별도의 변환 작업은 필요 없습니다. 시스템이 자동으로 각 문서에서 텍스트를 추출합니다. 하위 폴더를 만들어 정리해도 되며, 하위 폴더 안의 문서도 모두 자동으로 인식됩니다.

**2단계. Jenkins에 접속하여 DSCORE-Knowledge-Sync를 실행합니다.**

웹 브라우저에서 Jenkins( <http://localhost:8080> )에 접속한 후 로그인합니다. 대시보드에서 DSCORE-Knowledge-Sync를 찾아 클릭하고, 왼쪽 메뉴의 "지금 빌드"를 클릭합니다. 별도의 설정 입력 없이 바로 실행됩니다. 폴더에 넣어 둔 모든 문서가 자동으로 처리됩니다.

**3단계. 완료 후 AI 챗봇에서 확인합니다.**

빌드 이력에 파란색 원이 표시되면 학습이 완료된 것입니다. Dify 챗봇에서 "방금 학습시킨 기획서에 어떤 내용이 있어?" 같은 질문을 던져 보십시오. AI가 실제 문서 내용을 기반으로 답변하면 학습이 정상적으로 완료된 것입니다.

문서가 갱신되면 같은 절차를 반복하면 됩니다. 변경된 내용이 AI에 자동 반영됩니다.

---

### Job #2: 질문-답변 형태로 추가 학습시키기

Job #1이 교과서를 통째로 읽히는 것이라면, Job #2는 예상 시험 문제와 모범 답안을 미리 준비시키는 것입니다.

문서를 통째로 학습시키는 것만으로도 AI는 충분히 유용한 답변을 제공합니다. 하지만 같은 문서를 질문-답변 형태로도 추가 학습시키면, AI가 질문의 의도를 더 정확히 파악하고 핵심만 골라 답변하게 됩니다. 두 가지를 모두 실행하면 답변 정확도가 눈에 띄게 향상됩니다.

**준비할 것:** Job #1과 동일한 폴더에 문서가 이미 들어 있으면 됩니다. 별도의 파일 준비는 필요 없습니다.

**절차:**

**1단계.** Jenkins에 접속하여 대시보드에서 DSCORE-Knowledge-Sync-QA를 찾아 클릭합니다.

**2단계.** "지금 빌드"를 클릭합니다. Job #1과 마찬가지로 별도의 설정 입력 없이 바로 실행됩니다.

**3단계.** 빌드 이력에 파란색 원이 표시되면 완료입니다. AI 챗봇에서 질문을 던져 답변 품질이 향상되었는지 확인해 보십시오.

**권장 순서:** Job #1을 먼저 실행하여 문서 원문을 학습시킨 후, Job #2를 실행하여 질문-답변 형태의 학습을 추가하는 것이 가장 효과적입니다.

---

### Job #3: 소스 코드 구조 학습시키기

GitLab에 있는 프로젝트의 폴더 구조, 주요 설정 파일, 코드 내용을 AI에 학습시킵니다.

학습이 완료되면 다음과 같은 질문에 AI가 실제 코드를 참조하여 답변할 수 있습니다.

- "이 프로젝트는 어떤 기술 스택을 사용하고 있어?"
- "인증 모듈이 어디에 있어?"
- "결제 처리 로직의 흐름을 설명해 줘"

코드를 직접 열어 보지 않아도, AI에게 물어보는 것만으로 프로젝트 구조를 빠르게 파악할 수 있습니다.

**준비할 것:** 학습시키려는 GitLab 저장소의 주소

**절차:**

**1단계.** Jenkins에 접속하여 대시보드에서 DSCORE-Code-Knowledge-Sync를 찾아 클릭합니다.

**2단계.** "Build with Parameters"를 클릭하면 입력란이 나타납니다. REPO_URL 항목에 GitLab 저장소 주소를 입력합니다.

(예: <http://gitlab:8929/root/my-project.git>)

**3단계.** "빌드" 버튼을 클릭합니다. 빌드 이력에 파란색 원이 표시되면 완료입니다.

---

### Job #5: 웹사이트 내용 학습시키기

사내 위키나 기술 블로그 등 웹 페이지의 내용을 AI가 수집하여 학습합니다. 여러 페이지에 흩어져 있는 정보를 AI가 한꺼번에 읽어 들이므로, 이후 통합적인 답변을 받을 수 있습니다.

사내 위키에 유용한 정보가 많지만 어디에 무엇이 있는지 찾기 어려운 경우, 이 기능으로 위키 전체를 AI에 학습시키면 "위키에 이런 내용 있어?" 라고 물어보는 것만으로 원하는 정보를 찾을 수 있습니다.

**준비할 것:** 수집할 웹사이트의 주소

**절차:**

**1단계.** Jenkins에 접속하여 대시보드에서 DSCORE-Web-Knowledge-Sync를 찾아 클릭합니다.

**2단계.** "Build with Parameters"를 클릭하면 입력란이 나타납니다. ROOT_URL 항목에 수집하려는 웹사이트 주소를 입력합니다.

**3단계.** "빌드" 버튼을 클릭합니다. 수집 결과를 미리 확인하는 단계가 포함되어 있으므로, 중간에 승인 요청이 나타나면 내용을 검토한 후 승인합니다. 빌드 이력에 파란색 원이 표시되면 완료입니다.

---

## 이런 상황에서 활용해 보십시오

### 신입 사원 온보딩

프로젝트 문서와 코드를 한 번에 학습시켜 두면, 신입 사원이 AI에게 자유롭게 질문할 수 있습니다.

- "이 프로젝트에서 결제 모듈은 어떻게 동작해?"
- "회원 가입 절차가 기획서에 어떻게 정의되어 있어?"
- "이 프로젝트의 전체 폴더 구조를 설명해 줘"

같은 질문을 반복해서 받는 선임 개발자의 시간을 절약하면서도, 신입 사원의 이해 속도를 높일 수 있습니다. 질문하는 쪽도, 답변하는 쪽도 부담이 줄어듭니다.

### 기획서 기반 빠른 이해

300페이지짜리 요구사항 정의서를 학습시킨 후, "이 시스템의 사용자 권한 체계를 요약해 줘"라고 물으면 AI가 문서 전체를 훑어 핵심만 정리해 줍니다.

신규 기능 개발 전 기획 의도를 빠르게 파악하거나, 오래전에 작성된 문서의 내용을 다시 확인할 때 유용합니다. 문서를 처음부터 끝까지 다시 읽을 필요 없이, 궁금한 부분만 AI에게 물어보면 됩니다.

### 흩어진 지식의 통합 검색

PDF에 있는 설계서, GitLab에 있는 코드, 위키에 있는 기술 문서를 모두 학습시키면, AI가 이 모든 출처를 통합하여 답변합니다.

"인증 관련 코드는 어디에 있고, 설계서에는 뭐라고 적혀 있어?" 라는 질문에 코드와 문서를 동시에 참조한 답변을 받을 수 있습니다. 여러 곳을 직접 돌아다니며 정보를 조합할 필요가 없어집니다.

지금까지 "그건 어디에 있더라..." 하며 시간을 쓰셨다면, 이 기능으로 그 시간을 크게 줄일 수 있습니다.

### 프로젝트 인수인계

담당자가 바뀔 때 가장 어려운 것은 "문서에 적혀 있지 않은 맥락"을 전달하는 일입니다. 프로젝트의 모든 문서와 코드를 AI에 학습시켜 두면, 후임자가 언제든 AI에게 질문하여 프로젝트의 맥락을 파악할 수 있습니다.

"이 설정 값은 왜 이렇게 되어 있어?", "이 모듈은 어떤 문서를 기반으로 만들어졌어?" 같은 질문에도 AI가 관련 문서와 코드를 함께 참조하여 답변합니다. 완벽한 대체는 아니지만, 인수인계의 질을 크게 높여 줍니다.

### 기술 검토 및 의사 결정 지원

새로운 기능을 기획하거나 기술적 의사 결정을 내릴 때, "현재 프로젝트에서 비슷한 기능이 이미 구현되어 있어?", "기존 설계서에서 이 부분에 대해 언급한 적 있어?" 같은 질문으로 기존 자산을 빠르게 점검할 수 있습니다.

이미 있는 것을 다시 만들거나, 이전에 검토했던 사항을 다시 논의하는 비효율을 줄여 줍니다.

## 참고 사항

- 학습시킨 내용은 Dify 지식 저장소에 저장되며, 해당 AI 앱을 사용하는 모든 사용자가 조회할 수 있습니다. 학습시키는 내용의 범위를 사전에 팀 내에서 합의하시기 바랍니다.
- 사내 기밀 문서를 학습시키기 전에 보안 정책을 확인하십시오.
- 시스템 외부로 데이터가 전송되지 않습니다. 모든 처리는 사내 로컬 서버에서 이루어집니다.
- 한글(.hwp) 파일은 현재 지원하지 않습니다. PDF로 변환한 후 학습시켜 주십시오.
- 비밀번호가 설정된 PDF 파일은 내용을 추출할 수 없습니다. 비밀번호를 해제한 후 넣어 주십시오.
- 학습에 소요되는 시간은 문서의 수와 크기에 따라 달라지며, 일반적으로 수 분 이내에 완료됩니다.
