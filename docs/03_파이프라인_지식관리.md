# 지식 관리 파이프라인 (Job #1~#3, #5)

## 1. 개요

지식 관리 파이프라인은 프로젝트에 산재한 문서, 소스 코드, 웹 콘텐츠를 AI가 학습할 수 있는 형태로 변환하여, Dify Knowledge Base에 자동으로 적재하는 기능입니다.

이 파이프라인을 실행하면 AI가 프로젝트 고유의 맥락을 이해하게 되어, 일반론이 아닌 실제 프로젝트 기반의 답변을 제공할 수 있습니다.

---

## 2. 기능 설명

### 2.1. 학습 전후 비교

| 구분 | 학습 전 | 학습 후 |
| --- | --- | --- |
| AI의 지식 범위 | 인터넷에 공개된 일반 정보만 알고 있습니다 | 프로젝트의 문서, 코드 구조, 설정 파일 내용까지 파악합니다 |
| 답변의 구체성 | "일반적으로 JWT, 세션 기반 인증 방식이 있습니다" | "현재 프로젝트는 JwtTokenProvider.java에서 JWT 인증을 사용하고 있으며, 토큰 만료 시간은 3600초입니다" |
| 정보 통합 | 문서와 코드를 별도로 찾아봐야 합니다 | AI가 문서와 코드를 동시에 참조하여 통합된 답변을 제공합니다 |

### 2.2. 학습 가능한 대상

| 학습 대상 | 지원 형식 | 담당 Job |
| --- | --- | --- |
| 프로젝트 문서 | PDF, Word(.docx), Excel(.xlsx), PowerPoint(.pptx), 텍스트(.txt) | Job #1 (문서형), Job #2 (Q&A형) |
| 소스 코드 구조 | GitLab에 등록된 Git 저장소 | Job #3 |
| 웹 콘텐츠 | 사내 위키, 기술 블로그, 외부 참고 문서의 URL | Job #5 |

### 2.3. 각 Job의 역할

**Job #1: DSCORE-Knowledge-Sync (문서형 학습)**

원본 문서에서 텍스트를 추출하여 Dify Knowledge Base에 적재합니다. AI는 적재된 원문을 직접 검색하여 답변에 활용합니다. 문서의 특정 구절을 인용하거나, 표의 내용을 참조하는 등 원문 기반의 정확한 답변이 가능해집니다.

PDF 내부에 포함된 표와 차트는 자동으로 인식되어, 로컬 Vision 모델이 이를 텍스트로 변환합니다. 별도의 이미지 추출 작업은 필요하지 않습니다.

**Job #2: DSCORE-Knowledge-Sync-QA (Q&A형 학습)**

동일한 원본 문서를 기반으로 Dify가 질문-답변 쌍을 자동 생성하여 적재합니다. Job #1이 원문 참조 방식이라면, Job #2는 예상 질문에 대한 모범 답안을 사전에 준비하는 방식입니다. 두 가지를 모두 실행하면 AI의 답변 정확도가 향상됩니다.

**Job #3: DSCORE-Code-Knowledge-Sync (소스 코드 학습)**

GitLab에 등록된 Git 저장소를 분석하여, 프로젝트의 디렉터리 구조와 핵심 설정 파일(README, package.json, pom.xml 등)의 내용을 컨텍스트 문서로 생성한 후 Dify에 적재합니다. 이후 AI는 "이 프로젝트의 기술 스택은 무엇인가?", "인증 모듈은 어디에 위치해 있는가?" 같은 코드 구조 관련 질문에 답변할 수 있습니다.

**Job #5: DSCORE-Web-Knowledge-Sync (웹 콘텐츠 학습)**

지정된 URL의 웹 페이지를 자동으로 수집하여, 본문 텍스트를 추출하고 정제한 후 Dify에 적재합니다. 사내 위키, 기술 블로그, 외부 참고 문서 등 웹에 산재한 정보를 AI의 지식 범위에 포함시킬 수 있습니다.

## 3. 사용 절차

### 3.1. Job #1: 프로젝트 문서 학습

**1단계. 문서를 공유 폴더에 배치합니다.**

학습시킬 문서를 아래 경로에 복사합니다. 이 폴더에 넣은 파일은 Jenkins 컨테이너에서 자동으로 접근 가능합니다.

```
<PROJECT_ROOT>/data/knowledges/docs/org/
```

PDF, Word, Excel, PowerPoint, 텍스트 파일을 지원하며, 별도의 형식 변환은 필요하지 않습니다. 시스템이 자동으로 텍스트를 추출합니다.

**2단계. Jenkins에서 Job을 실행합니다.**

1. Jenkins(`http://localhost:8080`)에 접속하여 로그인합니다.
2. 대시보드에서 `DSCORE-Knowledge-Sync` Job을 선택합니다.
3. 좌측 메뉴의 `지금 빌드(Build Now)`를 클릭합니다.
4. 별도의 파라미터 입력 없이 즉시 실행됩니다.

**3단계. 학습 결과를 확인합니다.**

1. 빌드가 완료되면(좌측 Build History에서 파란색 아이콘 확인) Dify 웹 UI에 접속합니다.
2. Knowledge Base에서 해당 Dataset을 열어 문서가 등록되어 있는지 확인합니다.
3. Dify 챗봇에서 학습시킨 문서의 내용에 관한 질문을 던져 AI가 정상적으로 답변하는지 확인합니다.

### 3.2. Job #2: Q&A형 추가 학습

Job #1 실행 후 추가로 실행합니다. 별도의 문서 준비는 필요하지 않으며, Job #1에서 사용한 동일한 문서가 자동으로 처리됩니다.

1. Jenkins에서 `DSCORE-Knowledge-Sync-QA` Job을 선택합니다.
2. `지금 빌드(Build Now)`를 클릭합니다.

Q&A형 학습은 별도의 Dataset(qa_model)에 저장되므로, Job #1의 Dataset과 중복되지 않습니다.

### 3.3. Job #3: 소스 코드 구조 학습

1. Jenkins에서 `DSCORE-Code-Knowledge-Sync` Job을 선택합니다.
2. `Build with Parameters`를 클릭합니다.
3. `REPO_URL` 파라미터에 GitLab 저장소 주소를 입력합니다.
   - 형식: `http://gitlab:8929/<프로젝트경로>.git`
   - GitLab 웹 UI에서 해당 프로젝트 페이지의 `Clone` 버튼을 클릭하면 주소를 확인할 수 있습니다.
   - 호스트 부분은 `gitlab:8929`으로 입력합니다 (Jenkins 컨테이너 내부에서 접근하는 주소입니다).
4. `빌드` 버튼을 클릭합니다.

### 3.4. Job #5: 웹 콘텐츠 학습

1. Jenkins에서 `DSCORE-Web-Knowledge-Sync` Job을 선택합니다.
2. `Build with Parameters`를 클릭합니다.
3. `ROOT_URL` 파라미터에 수집할 웹 페이지의 주소를 입력합니다.
   - 입력한 페이지와 해당 페이지에서 링크된 하위 페이지가 함께 수집됩니다.
4. `빌드` 버튼을 클릭합니다.

외부 웹사이트를 수집할 경우, 해당 사이트의 이용 정책을 사전에 확인하십시오. 인증이 필요한 사내 시스템의 경우 자동 수집이 제한될 수 있으며, 이 경우 해당 내용을 수동으로 텍스트 파일에 저장한 후 Job #1을 통해 학습시키는 방법을 사용하십시오.

## 4. 권장 실행 순서

프로젝트에 처음으로 지식 베이스를 구축할 때는 다음 순서를 권장합니다.

| 순서 | 작업 | 비고 |
| --- | --- | --- |
| 1 | 문서를 공유 폴더에 배치 | 수동 작업 |
| 2 | Job #1 실행 (문서형 학습) | 필수 |
| 3 | Job #2 실행 (Q&A형 학습) | 권장 |
| 4 | Job #3 실행 (코드 구조 학습) | Git 저장소가 있는 경우 |
| 5 | Job #5 실행 (웹 콘텐츠 학습) | 참고할 웹 문서가 있는 경우 |
| 6 | Dify 챗봇에서 학습 결과 확인 | 수동 확인 |

이후에는 문서가 갱신되거나 코드가 변경될 때 해당 Job만 재실행하면 됩니다.

## 5. 활용 범위

| 활용 시나리오 | 설명 |
| --- | --- |
| 신규 인력 온보딩 | 프로젝트 문서와 코드를 학습시켜 두면, 신규 인력이 AI를 통해 프로젝트 구조와 업무 맥락을 빠르게 파악할 수 있습니다 |
| 기획서 요약 및 검색 | 대용량 요구사항 정의서를 학습시킨 후, "사용자 권한 체계를 요약해줘"와 같은 요청으로 핵심 내용을 빠르게 추출할 수 있습니다 |
| 분산된 지식의 통합 검색 | 문서, 코드, 위키를 모두 학습시키면 AI가 여러 출처를 동시에 참조하여 통합된 답변을 제공합니다 |
| 품질 분석 진단 품질 향상 | 지식 관리 파이프라인으로 프로젝트를 학습시킨 후 품질 분석 파이프라인(Job #4)을 실행하면, AI가 프로젝트 맥락에 맞는 구체적인 코드 진단을 수행합니다 |

---

## 6. 참고 사항

- 학습된 문서는 Dify Knowledge Base에 저장되며, 해당 Dify 앱을 사용하는 모든 사용자가 조회할 수 있습니다. 사내 기밀 문서를 학습시키기 전에 보안 정책을 확인하십시오.
- 모든 데이터 처리는 로컬 서버에서 이루어지며, 외부로 전송되지 않습니다.
- 동일한 파일명의 문서를 재실행하면 Dify에 중복 등록될 수 있습니다. 문서를 갱신할 때는 Dify에서 기존 문서를 삭제한 후 재실행하는 것을 권장합니다.
