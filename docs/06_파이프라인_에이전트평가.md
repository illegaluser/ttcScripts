# AI 에이전트 평가 파이프라인 (Job #7)

## 1. 개요

AI 에이전트 평가 파이프라인은 외부 AI 서비스(챗봇, API 등)에 시험 문제를 자동으로 출제하고, 응답을 11대 지표로 채점하여 성적표를 발행하는 기능입니다.

시험 문제(CSV 파일)를 준비하면 시스템이 자동으로 AI에게 질문을 전송하고, 각 응답을 객관적인 기준으로 채점하며, 결과를 시각적 성적표와 대시보드로 제공합니다.

---

## 2. 기능 설명

### 2.1. 11대 측정 지표

평가는 4단계 필터링 구조로 진행됩니다. 상위 단계에서 실패하면 하위 단계를 건너뛰어 리소스를 절약합니다.

#### 1단계: 기본 안전 검사 (Fail-Fast)

| 지표 | 검사 내용 |
| --- | --- |
| 개인정보 노출 검사 (Policy Violation) | 주민등록번호, 카드번호, 이메일, 전화번호 등 개인정보가 응답에 포함되는지 검사합니다. 하나라도 발견되면 즉시 불합격 처리되며 이후 채점은 진행하지 않습니다. |
| 응답 형식 검사 (Format Compliance) | API 방식의 AI가 약속된 응답 형식을 준수하는지 검사합니다. 웹 채팅 방식에는 적용되지 않습니다. |

#### 2단계: 과업 검사

| 지표 | 검사 내용 |
| --- | --- |
| 과업 달성도 (Task Completion) | 시험 문제에 명시된 성공 기준을 충족했는지 판정합니다. 키워드 포함 여부 등 규칙 기반 판정을 우선 수행하고, 자연어로 기술된 기준은 심판 AI가 판정합니다. |

#### 3단계: 답변 품질 심층 채점

| 지표 | 검사 내용 | 적용 대상 |
| --- | --- | --- |
| 질문 적합도 (Answer Relevancy) | 질문의 의도에 맞는 답변인지 채점합니다 | 모든 대상 |
| 유해성 (Toxicity) | 혐오 발언, 차별적 표현, 비속어가 포함되어 있는지 검사합니다 | 모든 대상 |
| 사실 근거 (Faithfulness) | 참고 자료에 근거한 답변인지, AI가 지어낸 내용인지 검사합니다 | RAG API만 |
| 정보 검색력 (Contextual Recall) | 답변에 필요한 정보를 올바르게 검색했는지 채점합니다 | RAG API만 |
| 검색 정밀도 (Contextual Precision) | 검색된 자료에 불필요한 정보가 섞여 있는지 채점합니다 | RAG API만 |

#### 4단계: 대화 능력 검사

| 지표 | 검사 내용 | 적용 조건 |
| --- | --- | --- |
| 대화 일관성 (Multi-turn Consistency) | 여러 차례 대화 시 이전 내용을 기억하고 일관된 답변을 유지하는지 채점합니다 | 2턴 이상 대화 |

#### 운영 지표 (상시 기록)

| 지표 | 기록 내용 |
| --- | --- |
| 응답 속도 (Latency) | 질문 전송부터 답변 수신까지 소요된 시간(밀리초)입니다 |
| 사용량 (Token Usage) | API 호출 시 소비된 토큰 수입니다. 비용 추정에 활용됩니다 |

### 2.2. 평가 대상 유형

| 유형 | 설명 |
| --- | --- |
| HTTP API | REST API 방식으로 통신하는 AI 서비스입니다 |
| 웹 UI 챗봇 | 브라우저에서 접속하는 채팅 인터페이스입니다. Playwright가 브라우저를 열어 채팅창에 직접 질문을 입력하고 응답을 추출합니다 |

### 2.3. 심판 모델

채점은 로컬에서 실행되는 AI(Ollama)가 수행합니다. 필요에 따라 외부 AI(OpenAI, Google Gemini)를 심판으로 사용할 수도 있습니다.

| 모드 | 설명 |
| --- | --- |
| local_ollama_wrapper | 로컬 Ollama 모델을 심판으로 사용합니다 (기본값, 비용 0원) |
| openai_wrapper | OpenAI API를 심판으로 사용합니다 |
| gemini_wrapper | Google Gemini API를 심판으로 사용합니다 |

## 3. 사용 절차

### 3.1. 시험 문제(golden.csv) 작성

시험 문제는 CSV 파일로 작성합니다. Excel 또는 텍스트 편집기에서 작성한 후 CSV(UTF-8) 형식으로 저장하십시오.

기본 구성은 다음과 같습니다.

| 컬럼 | 필수 여부 | 설명 |
| --- | --- | --- |
| case_id | 필수 | 테스트 케이스의 고유 식별자입니다 |
| input | 필수 | AI에게 던질 질문입니다 |
| expected_output | 선택 | 기대하는 정답입니다 |
| success_criteria | 선택 | 성공 판정 기준입니다 |

성공 기준은 다음과 같은 형태로 작성할 수 있습니다.

| 작성 형태 | 예시 |
| --- | --- |
| 키워드 포함 | 응답에 '서울'이 포함되어야 함 |
| 복합 조건 | 응답에 '서울' 또는 'Seoul'이 포함되어야 함 |
| 자연어 기준 | 답변이 공손하고 정확해야 한다 (심판 AI가 판정) |

여러 차례 대화를 테스트하려면 `conversation_id`와 `turn_id` 컬럼을 추가합니다. 같은 `conversation_id`를 가진 행이 하나의 대화로 묶이며, `turn_id` 순서대로 실행됩니다.

효과적인 평가를 위해 기능 테스트(5~10건), 안전성 테스트(3~5건), 대화 테스트(2~3건) 등 다양한 유형을 포함하여 최소 10건 이상의 시험 문제를 작성하는 것을 권장합니다.

### 3.2. Jenkins에서 평가 실행

1. Jenkins(`http://localhost:8080`)에서 `DSCORE-Agent-Eval` Job을 선택합니다.
2. `Build with Parameters`를 클릭합니다.
3. 다음 파라미터를 입력합니다.

| 파라미터 | 설명 | 입력 예시 |
| --- | --- | --- |
| `TARGET_URL` | 평가 대상 AI의 접속 주소입니다 | `http://host.docker.internal:8000/invoke` |
| `TARGET_TYPE` | 평가 방식입니다. `http`(API)와 `ui_chat`(웹 채팅) 중 선택합니다 | `http` |
| `TARGET_MODE` | 연결 방식입니다. 로컬 AI는 `local_ollama_wrapper`, 외부 AI는 `direct`를 선택합니다 | `local_ollama_wrapper` |
| `UPLOADED_GOLDEN_DATASET` | 작성한 시험 문제 파일을 업로드합니다 | (파일 선택) |

4. `빌드` 버튼을 클릭합니다.

### 3.3. 결과 확인

빌드 완료 후 다음 경로에서 결과를 확인할 수 있습니다.

| 확인 위치 | 방법 |
| --- | --- |
| Jenkins AI Eval Summary 탭 | 빌드 페이지 상단의 `AI Eval Summary` 탭을 클릭하면 시각적 성적표가 표시됩니다 |
| Jenkins Artifacts | 빌드 페이지의 `Artifacts`에서 `summary.html`을 클릭하면 상세 성적표를 브라우저에서 확인할 수 있습니다. 이 파일은 다운로드하여 동료에게 공유할 수 있습니다 |
| Langfuse 대시보드 | `http://localhost:3000`에 접속하면 평가 추이를 시계열 그래프로 모니터링할 수 있습니다. `Traces` 메뉴에서 각 질문/답변/채점 과정의 상세 기록을, `Scores` 메뉴에서 지표별 점수 분포를 확인할 수 있습니다 |

Langfuse는 선택 사항이며, 설정되어 있지 않아도 Jenkins 아티팩트를 통해 성적표를 확인할 수 있습니다.

## 4. 활용 범위

| 활용 시나리오 | 설명 |
| --- | --- |
| 신규 AI 모델 도입 전 성능 비교 | 기존 모델과 신규 모델에 동일한 시험 문제를 적용하여 객관적으로 성능을 비교할 수 있습니다 |
| 고객 응대 챗봇 정기 품질 점검 | 고객이 자주 묻는 질문으로 시험지를 구성하여 정기적으로 챗봇의 답변 품질을 모니터링할 수 있습니다 |
| AI 보안 검증 | 개인정보 노출, 유해 발언 등 안전성 관련 시험 문제로 AI의 보안 수준을 검증할 수 있습니다 |
| 경영진 보고용 객관적 수치 확보 | "답변 정확도 92%, 유해성 0건, 평균 응답 시간 1.2초" 와 같은 정량적 성과 지표를 산출할 수 있습니다 |
| 여러 AI 비교 평가 | 동일한 시험지로 여러 AI를 평가하여 성능 순위를 산출할 수 있습니다 |

---

## 5. 참고 사항

- 시험 문제는 최소 10건 이상, 기능 / 안전성 / 대화 등 다양한 유형을 포함하는 것을 권장합니다.
- 채점은 로컬 AI가 수행하므로 시험 내용이 외부로 유출되지 않습니다.
- CSV 파일은 반드시 UTF-8 인코딩으로 저장하십시오. Excel에서 저장할 경우 "CSV UTF-8" 형식을 선택합니다.
- Langfuse 없이도 성적표(summary.html) 확인이 가능합니다. Langfuse는 장기 추이 모니터링이 필요할 때 활용하십시오.
