# DSCORE-TTC 프레임워크 개요

## 1. 개요

DSCORE-TTC(TTC)는 **외부 네트워크가 차단된 환경에서 AI를 활용하여 소프트웨어 개발의 생산성과 품질 관리를 자동화하는 온프레미스(On-premise) QAOps 프레임워크**입니다.

본 프레임워크는 민감한 내부 데이터의 외부 유출 없이, AI 기술의 이점을 개발 과정에 적용하기 위해 설계되었습니다. 
모든 기능은 외부 서비스 호출 없이 단일 로컬 머신 내에서 완결적으로 동작하여, 금융, 국방, 공공 등 높은 수준의 보안이 요구되는 환경에서도 안전하게 사용할 수 있습니다.

---

## 2. 해결하고자 하는 과제

소프트웨어 개발 및 유지보수 과정에서는 반복적으로 발생하는 비효율과 병목 현상이 존재합니다. 
TTC 프레임워크는 다음 다섯 가지 핵심 과제를 해결하는 데 중점을 둡니다.

### 과제 1: 지식의 파편화 및 중앙화 부재

프로젝트 관련 정보(기획서, 설계서, 소스 코드, 기술 자료 등)가 여러 시스템과 개인의 PC에 분산되어 있어, 신규 인력이 프로젝트의 전체 맥락을 파악하는 데 많은 시간이 소요됩니다. 
정보 접근성이 낮아 팀 전체의 생산성을 저해하는 요인이 됩니다.

### 과제 2: 코드 품질 관리의 수동적 프로세스

정적 분석 도구(SonarQube 등)가 코드의 잠재적 결함을 발견하더라도, 각 이슈의 유효성을 판단하고, 원인을 분석하며, 이슈 관리 시스템에 등록하는 과정은 상당 부분 수작업에 의존합니다. 
이 과정에 많은 공수가 투입되어, 개발자가 핵심 개발 활동에 집중하기 어렵게 만듭니다.

### 과제 3: E2E(End-to-End) 테스트의 높은 유지보수 비용

사용자 시나리오를 검증하는 E2E 테스트는 품질 확보에 필수적이나, 웹사이트의 UI가 일부만 변경되어도 테스트 코드가 쉽게 실패하는 '깨지기 쉬운 테스트(Brittle Test)' 문제가 고질적으로 발생합니다. 
이로 인해 테스트 코드의 유지보수 비용이 과도하게 증가하여, 테스트의 지속가능성이 위협받습니다.

### 과제 4: 보안 제약으로 인한 최신 AI 기술 도입의 어려움

최신 AI 모델은 개발 생산성을 크게 향상시킬 수 있는 잠재력을 가졌지만, 대부분 외부 클라우드 서비스 형태로 제공됩니다. 내부 코드나 데이터를 외부 서버로 전송하는 것이 금지된 보안 규정으로 인해, 이러한 AI 기술의 도입 자체가 원천적으로 차단됩니다.

### 과제 5: AI 모델 성능 평가의 주관성 및 비효율성
AI 챗봇이나 LLM 기반 서비스의 성능을 평가하는 작업은 주로 수동 테스트에 의존하여, 평가자의 주관이 개입될 여지가 많고 일관된 기준을 적용하기 어렵습니다. 
또한, 모델이 업데이트될 때마다 동일한 품질을 보장하기 위한 회귀 테스트에 많은 시간이 소요되어, AI 서비스의 품질을 객관적으로 관리하고 지속적으로 개선하는 데 어려움이 있습니다.

## 3. 핵심 기능 및 기대 효과

DSCORE-TTC는 상기 5대 과제를 해결하기 위해 AI 기반 자동화 파이프라인을 제공하며, 이를 통해 다음과 같은 효과를 기대할 수 있습니다.

### 기능 1: 중앙화된 AI 지식 베이스 구축 (지식 관리 파이프라인)

- **설명:** 프로젝트와 관련된 모든 형태의 문서(PDF, Word 등)와 소스 코드, 웹 콘텐츠를 시스템에 등록하면, AI가 해당 내용을 학습하여 중앙화된 지식 베이스를 구축합니다.
- **기대 효과:** 팀원은 흩어진 정보를 찾아 헤맬 필요 없이, 자연어 질의를 통해 필요한 정보를 즉시 얻을 수 있습니다. 신규 인력의 온보딩 기간이 단축되고, 팀 내 정보 격차가 해소됩니다.

### 기능 2: 코드 품질 분석 및 이슈 등록 자동화 (품질 분석 파이프라인)

- **설명:** 정적 분석 도구의 분석 결과를 AI가 자동으로 해석하고, 프로젝트의 맥락을 고려하여 발견된 이슈의 원인과 수정 방안을 제시합니다. 분석된 내용은 이슈 관리 시스템(GitLab)에 자동으로 등록됩니다.
- **기대 효과:** 품질 분석 리포트를 검토하고 이슈를 수동으로 생성하는 반복적인 작업을 제거하여, 개발자가 고부가가치 활동에 집중할 수 있도록 지원합니다.

### 기능 3: 자연어 기반 E2E 테스트 자동화 (자율 E2E 테스트 파이프라인)

- **설명:** 테스트 담당자가 자연어(한글)로 테스트 시나리오를 작성하면, AI가 이를 해석하여 실제 브라우저를 구동하는 테스트로 자동 변환 및 실행합니다. UI 요소가 변경되더라도 AI가 의미적으로 요소를 재탐색하는 '자동 복원(Self-healing)' 기능이 포함되어 있습니다.
- **기대 효과:** 테스트 코드 작성에 대한 부담을 제거하고, UI 변경에 강건한 테스트를 운영하여 E2E 테스트의 유지보수 비용을 획기적으로 절감합니다.

### 기능 4: 폐쇄망 환경에서의 AI 기능 제공 (로컬 LLM 기반 아키텍처)

- **설명:** 프레임워크의 모든 AI 기능은 Ollama를 통해 로컬 머신에서 직접 구동되는 LLM(거대 언어 모델)을 기반으로 동작합니다.
- **기대 효과:** 외부 인터넷 연결 없이 모든 기능을 사용할 수 있어, 데이터 유출 우려 없이 엄격한 보안 환경에서도 AI를 활용한 개발 프로세스 혁신이 가능합니다.

### 기능 5: AI 에이전트 성능의 정량적 평가 자동화 (AI 에이전트 평가 파이프라인)
- **설명:** 사전에 정의된 평가 데이터셋을 기반으로 AI 모델의 응답을 자동으로 수집하고, 답변 정확도, 유해성, 일관성 등 다차원적인 지표에 따라 성능을 정량적으로 측정하고 리포트를 생성합니다.
- **기대 효과:** AI 모델의 품질을 객관적인 데이터로 관리할 수 있게 되며, 모델 업데이트 전후의 성능 변화를 정확하게 비교 분석하여 데이터 기반의 의사결정을 지원합니다. 수동 평가에 소요되는 비용과 시간을 크게 절감합니다.

## 4. 주요 활용 대상

DSCORE-TTC는 개발 프로세스에 참여하는 다양한 역할의 구성원에게 다음과 같은 가치를 제공합니다.

- **기획자 / 제품 책임자(PO):** 요구사항 명세서나 기획서를 시스템에 업로드하여, 해당 기능이 의도대로 동작하는지를 테스트 시나리오로 자동 생성하고 검증할 수 있습니다.
- **QA 엔지니어:** 코딩 기술이 없어도 자연어로 테스트 케이스를 작성하고, 이를 기반으로 회귀 테스트, 스모크 테스트 등 다양한 E2E 테스트를 자동화할 수 있습니다.
- **소프트웨어 개발자:** AI 지식 베이스를 통해 프로젝트 구조나 과거의 의사결정 기록을 신속하게 파악할 수 있으며, 코드 품질 문제를 자동으로 리포팅 받아 개발에만 집중할 수 있습니다.
- **팀 리드 / 관리자:** 개발 중인 AI 모델이나 챗봇 서비스의 성능을 객관적인 지표로 상시 측정하고, 품질 변화 추이를 정량적으로 파악하여 데이터 기반의 의사결정을 지원합니다.

## 5. 프레임워크 구조

DSCORE-TTC는 5개의 단계로 구성되어 있으며, 각 단계는 독립적으로, 또는 전체를 통합하여 유기적으로 운영할 수 있습니다.

```
┌──────────────────────────────────────────────────┐
│             Phase 1: DevOps 인프라 구축              │
│  (Jenkins, SonarQube, GitLab, Dify, Langfuse)    │
└─────────────────────┬────────────────────────────┘
                      │
                      ▼
┌──────────────────────────────────────────────────┐
│             Phase 2: 지식 관리 (RAG)               │
│ (문서, 코드, 웹 콘텐츠를 AI에게 학습) → [AI 지식 베이스]│
└─────────────────────┬────────────────────────────┘
                      │
                      ▼
┌──────────────────────────────────────────────────┐
│             Phase 3-5: AI 자동화 파이프라인          │
│                                                  │
│  [품질 분석]  [자율 E2E 테스트]  [AI 에이전트 평가]  │
│  (SonarQube    (자연어/기획서        (시험문제 CSV     │
│   → AI 진단     → 테스트 자동화)     → 11대 지표 채점) │
│   → GitLab 이슈)                                  │
└──────────────────────────────────────────────────┘
```

| 단계 | 공식 명칭 | 상세 설명 |
| --- | --- | --- |
| **Phase 1** | DevOps 인프라 | 자동화 파이프라인 구동에 필요한 Jenkins, SonarQube 등 기반 도구를 컨테이너 환경에 구축합니다. |
| **Phase 2** | 지식 관리 | 분산된 프로젝트 관련 지식 자산을 수집하여 AI가 검색 및 추론할 수 있는 형태로 변환 및 저장합니다. |
| **Phase 3** | 품질 분석 | 코드 정적 분석 결과를 AI가 진단하고, 이슈 관리 시스템에 수정 요청을 자동으로 등록합니다. |
| **Phase 4** | 자율 E2E 테스트 | 자연어 요구사항을 기반으로 E2E 테스트 시나리오를 자동 생성하고 브라우저에서 실행합니다. |
| **Phase 5** | AI 에이전트 평가 | 개발된 AI 모델 또는 챗봇의 성능을 정량적 지표에 따라 자동으로 평가하고 리포트합니다. |

프레임워크를 구성하는 상세 기술 요소 및 아키텍처에 대한 정보는 `02_시스템_아키텍처.md` 문서를 참고해 주시기 바랍니다.
