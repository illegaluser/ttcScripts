# DSCORE-TTC 시스템 아키텍처

## 1. 배포 구조: 단일 머신, 폐쇄망 대응

DSCORE-TTC의 모든 서비스는 **개발자의 로컬 머신(맥북 / 윈도우) 한 대** 위에서 구동됩니다. 
외부 서버, 클라우드 인프라, 인터넷 연결이 필요하지 않습니다.

```text
┌─────────────────────────────────────────────────────────────────────┐
│                            단일 로컬 머신                              │
│                                                                     │
│   ┌───────────────────────────────────────────────────────────┐     │
│   │              Docker Compose (devops-net)                  │     │
│   │                                                           │     │
│   │   Jenkins    SonarQube    GitLab    Langfuse    DB x2     │     │
│   │   :8080      :9000        :8929     :3000                 │     │
│   └───────────────────────────────────────────────────────────┘     │
│                          │                                          │
│   ┌────────────────┐     │     ┌──────────────────────────────┐     │
│   │  Dify          │◄────┘     │  Ollama (로컬 LLM 엔진)        │     │
│   │  (별도 Compose) │           │  :11434                      │     │
│   │  AI 워크플로우  │           │  qwen3-coder, llama3.2-vision  │     │
│   └────────────────┘           └──────────────────────────────┘     │
│                                                                     │
│   ※ 외부 인터넷 연결 불필요. 모든 데이터가 이 머신 안에서 처리됩니다.              │
└─────────────────────────────────────────────────────────────────────┘
```

### 폐쇄망에서의 동작 원리

| 구성 요소 | 외부 연결 필요 여부 | 설명 |
| --- | --- | --- |
| Jenkins, SonarQube, GitLab | 불필요 | Docker 이미지를 사전에 다운로드하여 로컬에 보관합니다. 이후 인터넷 없이 구동 가능합니다. |
| Dify | 불필요 | 자체 Docker Compose로 로컬 구동합니다. |
| Ollama (LLM) | 최초 모델 다운로드 시에만 필요 | 모델을 한 번 다운로드하면 이후 인터넷 없이 무제한 사용 가능합니다. 추론 비용은 0원입니다. |
| Langfuse | 불필요 | 로컬 PostgreSQL에 데이터를 저장합니다. |
| 외부 AI (OpenAI, Gemini) | **선택 사항** | 개방망 환경에서 성능을 높이고 싶을 때만 사용합니다. 폐쇄망에서는 완전히 생략 가능합니다. |

### 개방망 vs 폐쇄망 구성 비교

| 항목 | 개방망 (인터넷 가능) | 폐쇄망 (인터넷 차단) |
| --- | --- | --- |
| LLM 추론 | 로컬 Ollama + 외부 AI(OpenAI/Gemini) 병행 가능 | 로컬 Ollama만 사용 |
| AI 응답 평가 | 로컬 모델 또는 GPT-4o 등 고성능 외부 모델 선택 가능 | 로컬 모델(qwen3-coder 등)만 사용 |
| Docker 이미지 | 빌드 시 자동 다운로드 | 사전에 이미지를 `docker save/load`로 반입해야 합니다. |
| Ollama 모델 | `ollama pull`로 즉시 다운로드 | 사전에 모델 파일을 USB 등으로 반입해야 합니다. |

│        │                     │                     │                    │
└────────┼─────────────────────┼─────────────────────┼────────────────────┘
         │                     │                     │

                                     │
                              ┌──────┴──────┐

                              │ :11434      │

                              │ (호스트 실행)  │

| 서비스 | 접속 URL | 역할 | 데이터 저장 위치 |
| --- | --- | --- | --- |
| **Jenkins** | `http://localhost:8080` | 파이프라인 실행 엔진. 모든 자동화 작업의 시작점 | `data/jenkins/` |
| **SonarQube** | `http://localhost:9000` | 소스 코드 정적 분석 결과 확인 | `data/sonarqube/` |
| **GitLab** | `http://localhost:8929` | 소스 코드 호스팅 및 이슈 트래커 | `data/gitlab/` |
| **Dify** | `http://localhost` (기본) | AI 워크플로우 설계 및 지식 베이스(RAG) 관리 | `dify/docker/volumes/` |
| **Langfuse** | `http://localhost:3000` | LLM 호출 관측성 및 AI 평가 결과 대시보드 | `data/postgres-langfuse/` |
| **Ollama** | `http://localhost:11434` | (API) 로컬 LLM 실행 엔진. | 호스트 `~/.ollama/` |

## 4. 주요 Jenkins 파이프라인

모든 자동화 워크플로우는 Jenkins 파이프라인(Job)으로 구현되어 있으며, 각 파이프라인의 실행은 파라미터를 통해 제어할 수 있습니다.

| 기능 분류 | 파이프라인 이름 | 한 줄 설명 |
| --- | --- | --- |
| **지식 관리** | DSCORE-TTC 지식주입 (문서 단순 청깅 및 임베딩) | 문서(PDF/Word/Excel)를 AI 지식 베이스에 업로드합니다. |
| | DSCORE-TTC 지식주입 (질문 및 답변 사전생성) | 문서를 Q&A 형태로 변환하여 AI 답변 품질을 높입니다. |
| | DSCORE-TTC 코드 사전학습 | Git 저장소의 코드 구조와 내용을 분석하여 AI가 학습합니다. |
| | DSCORE-TTC 웹스크래핑 | 웹사이트를 크롤링하여 최신 정보를 지식 베이스에 추가합니다. |
| **품질 분석** | DSCORE-TTC 코드 정적분석 | SonarQube를 통해 소스 코드의 품질 이슈를 탐지합니다. |
| | DSCORE-TTC 코드 정적분석 결과분석 및 이슈등록 | 탐지된 품질 이슈를 AI가 분석하고 GitLab에 이슈를 등록합니다. |
| **자율 테스트** | DSCORE-ZeroTouch-QA | 자연어 시나리오 기반으로 E2E 테스트를 자율 수행합니다. |
| **AI 평가** | DSCORE-TTC AI평가 | 외부 AI 에이전트의 성능을 11대 지표로 자동 평가합니다. |

각 파이프라인의 상세한 사용 방법은 개별 가이드 문서(03~06)를 참조하십시오.

## 5. 네트워크 구성 (devops-net)

모든 컨테이너 서비스는 `devops-net`이라는 Docker 가상 네트워크로 연결됩니다. 이를 통해 서비스 간에 컨테이너 이름(예: `sonarqube`, `gitlab`)으로 직접 통신할 수 있습니다.

| 통신 경로 | 사용되는 URL | 설명 |
| --- | --- | --- |
| Jenkins → SonarQube | `http://sonarqube:9000` | 품질 분석 파이프라인에서 정적 분석 결과를 API로 조회 |
| Jenkins → GitLab | `http://gitlab:8929` | 품질 분석 파이프라인에서 이슈를 API로 생성 |
| Jenkins → Dify | `http://api:5001/v1` | 지식 관리/품질 분석 파이프라인에서 지식 업로드 및 워크플로우 실행 |
| Jenkins → Langfuse | `http://langfuse-server:3000` | AI 평가 파이프라인에서 평가 결과를 전송 |
| Jenkins → Ollama | `http://host.docker.internal:11434` | 로컬 LLM 호출 (호스트 머신 경유) |

Dify는 별도의 Docker Compose로 관리되며, `docker-compose.override.yaml`을 통해 `devops-net`에 연결됩니다.

## 6. 보안 및 데이터 주권

DSCORE-TTC에서 처리되는 모든 데이터의 흐름은 다음과 같습니다.

| 데이터 종류 | 저장 위치 | 외부 전송 여부 |
| --- | --- | --- |
| 프로젝트 문서 (PDF, Word 등) | `data/knowledges/docs/` | 전송하지 않음 |
| 소스 코드 | GitLab 컨테이너 (`data/gitlab/`) | 전송하지 않음 |
| 정적 분석 결과 | SonarQube 컨테이너 (`data/sonarqube/`) | 전송하지 않음 |
| AI 학습 데이터 (Knowledge Base) | Dify 컨테이너 내부 볼륨 | 전송하지 않음 |
| AI 평가 결과 | Langfuse DB (`data/postgres-langfuse/`) | 전송하지 않음 |
| LLM 추론 (질문/답변) | Ollama (호스트 메모리) | 전송하지 않음 |
| E2E 테스트 스크린샷 | Jenkins 아티팩트 (`artifacts/`) | 전송하지 않음 |

> 외부 AI 서비스(OpenAI, Gemini 등)를 **선택적으로** 연동한 경우에만, 해당 서비스에 질문 데이터가 전송됩니다. 폐쇄망 환경에서는 이 연동을 사용하지 않으므로 데이터 유출 경로가 존재하지 않습니다.

## 7. 폐쇄망 도입 시 사전 반입 체크리스트

인터넷이 차단된 환경에서 시스템을 구축하려면, 다음 항목을 사전에 반입해야 합니다.

| 반입 항목 | 용량 (참고) | 반입 방법 |
| --- | --- | --- |
| Docker 이미지 6종 (Jenkins, SonarQube, GitLab, Langfuse, PostgreSQL x2) | 약 10~15GB | `docker save` → USB → `docker load` |
| Dify Docker 이미지 | 약 3~5GB | 동일 |
| Ollama 설치 파일 (macOS) | 약 100MB | USB |
| LLM 모델 파일 (qwen3-coder:30b, llama3.2-vision 등) | 모델당 5~20GB | USB → `ollama` 모델 디렉터리에 복사 |
| 프로젝트 소스 코드 (본 리포지토리) | 약 500MB | USB → Git clone 또는 직접 복사 |

## 8. 전체 기술 스택

| 구분 | 기술 | 역할 |
| --- | --- | --- |
| CI/CD | Jenkins | 파이프라인 실행 및 스케줄링 |
| 코드 품질 | SonarQube | 정적 코드 분석 (버그, 취약점, 코드 스멜 탐지) |
| 소스 관리 | GitLab | Git 저장소 호스팅 및 이슈 트래커 |
| AI 플랫폼 | Dify | LLM 워크플로우 설계, 지식 베이스(RAG), Chatflow |
| AI 관측성 | Langfuse | LLM 호출 추적, 평가 점수 대시보드, 비용 모니터링 |
| 로컬 LLM | Ollama | 온프레미스 LLM 실행 엔진 (qwen3-coder 등) |
| E2E 테스트 | Playwright | 브라우저 자동화 엔진 |
| AI 평가 | DeepEval + Promptfoo | LLM 응답 심층 채점 및 보안 검사 |
| 인프라 | Docker Compose | 전체 서비스의 컨테이너화 및 네트워크 구성 |
