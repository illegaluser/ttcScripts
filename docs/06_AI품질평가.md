# AI 품질 평가 파이프라인

## 1. 파이프라인의 목적

AI 챗봇이나 LLM 기반 서비스를 도입하거나 운영할 때, 해당 AI의 품질과 성능을 객관적으로 측정하고 관리하는 것은 매우 중요한 과제입니다. 그러나, 사람이 직접 질문하고 주관적으로 답변 품질을 판단하는 수동적인 방식은 일관성이 부족하고 일부 케이스만 확인할 수 있다는 명백한 한계를 가집니다.

AI 에이전트 평가 파이프라인은 이러한 과정을 자동화하고, 데이터 기반의 객관적인 평가 체계를 제공하는 것을 목적으로 합니다. 
사전에 정의된 평가 데이터셋을 사용하여 AI 에이전트의 답변을 수집하고, 다차원적인 기준에 따라 정량적으로 평가하여 시각적인 리포트를 생성합니다. 
이를 통해 수동 평가에 소요되는 비용을 절감하고, AI의 품질 변화를 지속적으로 추적 및 관리할 수 있습니다.

---

## 2. 동작 흐름

```
[평가 데이터셋 (CSV)] ──(1. 정의/2. 실행)──▶ [평가 대상 AI]
  - 질의 (Prompt)
  - 평가 기준 (Criteria)                  (응답 회신)
                                             │
                                             ▼
[평가 리포트 (HTML)] ◀──(3. 평가/측정)───── [AI 평가 시스템]
  - 11대 지표 결과                       - 단계 1: 안전성 검사 (PII 등)
                                         - 단계 2: 요구사항 충족도 검사
                                         - 단계 3: 응답 품질 심층 평가 (LLM)
                                         - 단계 4: 다중 대화 일관성 평가
```

이 파이프라인은 **"평가 케이스 정의 → 평가 실행 → 응답 평가 및 측정 → 리포트 생성"** 의 4단계 흐름으로 동작합니다.

### 단계 1: 평가 케이스 정의

사용자는 평가의 기준이 될 데이터셋을 준비합니다. 
평가 데이터셋은 "AI에게 전달할 질의(Prompt)"와 "해당 질의에 대한 답변이 만족해야 할 성공 기준(Criteria)"으로 구성된 목록이며, CSV 파일 형식으로 작성됩니다. 
성공 기준은 단순 키워드 포함 여부부터 정규식을 사용한 복잡한 패턴 검증까지 다양하게 설정할 수 있습니다. 
이를 통해 기능적 정확성, 안전성(개인정보 비노출 등), 대화의 일관성 등 다양한 측면을 검증하는 케이스를 설계할 수 있습니다.

### 단계 2: 평가 실행

시스템은 준비된 평가 데이터셋의 케이스를 순차적으로 실행하며, 각 질의를 평가 대상 AI 에이전트에게 전송하고 응답을 수집합니다. 
평가 대상 에이전트의 형태에 따라 직접적인 API 호출 방식과, 실제 사용자와 같이 웹 브라우저의 채팅 UI를 통해 상호작용하는 방식을 모두 지원합니다.

### 단계 3: 응답 평가 및 측정

수집된 각 응답은 11가지의 세부 지표에 따라 자동으로 평가됩니다. 
평가는 아래와 같은 4단계의 필터링 구조로 효율적으로 진행되며, 상위 단계의 필수 기준을 통과하지 못한 경우 불필요한 하위 단계 평가는 생략됩니다.

-   **1단계 — 기본 안전성 검사:** 응답에 개인정보(주민등록번호, 카드번호 등)나 사전에 정의된 금칙어가 포함되어 있는지 여부를 검사합니다.
-   **2단계 — 요구사항 충족도 검사:** 평가 케이스에 명시된 성공 기준(예: "답변에 '서울'이라는 단어가 반드시 포함되어야 함")을 충족했는지 확인합니다.
-   **3단계 — 응답 품질 심층 평가:** 평가 LLM(Evaluator LLM)이 평가자로 기능하여, 답변이 질의의 의도에 부합하는지(Answer Relevancy), 유해하거나 공격적인 표현이 없는지(Toxicity), 사실에 근거하지 않은 내용을 지어내지 않았는지(Faithfulness) 등 답변의 질적인 측면을 심층적으로 평가합니다.
-   **4. 단계 4 — 다중 대화 일관성 평가:** 여러 차례의 질의응답으로 구성된 시나리오의 경우, AI가 이전 대화의 맥락을 올바르게 기억하고 일관된 답변을 생성하는지를 종합적으로 평가합니다.

이 외에도 각 응답의 생성에 소요된 시간(Latency)과 처리량(Throughput)이 자동으로 측정되어, AI의 성능 지표로 활용됩니다.

### 단계 4: 평가 리포트 생성

모든 평가가 완료되면, 각 케이스별 상세 결과와 전체 요약 통계를 포함하는 시각적인 HTML 리포트가 생성됩니다. 
리포트에는 각 질의에 대한 AI의 실제 응답과 모든 평가지표별 통과/실패 여부가 명확하게 기록되어 있어, 심층적인 결과 분석을 지원합니다. 
이 리포트는 Jenkins 빌드 결과 페이지에서 즉시 확인하거나 파일로 다운로드할 수 있습니다.

## 3. 평가 지표 선정 기준 및 상세

본 파이프라인은 AI 에이전트의 성능을 다각적으로 측정하기 위해, 산업계의 LLM 평가 연구(RAG, LLM-as-a-judge 등)에서 널리 통용되는 기준들을 기반으로 11대 핵심 지표를 선정하였습니다. 지표 선정 시 다음 네 가지 원칙을 고려하였습니다.

-   **포괄성 (Comprehensiveness):** 답변의 품질, 정확성, 안전성, 일관성 등 AI 에이전트가 갖춰야 할 핵심 역량을 전반적으로 평가할 수 있어야 합니다.
-   **자동화 가능성 (Automatability):** 평가 과정에 사람의 주관적 개입을 최소화하고, 평가 LLM 및 자동화된 규칙을 통해 일관되고 반복적으로 측정할 수 있어야 합니다.
-   **업무 관련성 (Business Relevance):** 실제 비즈니스 환경에서 AI 서비스의 품질을 판단하는 데 실질적으로 중요한 요소들을 중심으로 구성되어야 합니다.
-   **객관성 (Objectivity):** 평가 결과가 정량적인 수치로 명확하게 표현되어, 모델 간의 성능 비교나 품질 변화 추이 분석이 용이해야 합니다.

### 11대 핵심 평가지표

| 구분 | 지표 (Metric) | 설명 |
| :--- | :--- | :--- |
| **정확성** | **1. 답변의 질문 관련성 (Answer Relevancy)** | 생성된 답변이 사용자의 질문 의도와 얼마나 관련이 있는지를 평가합니다. |
| | **2. 답변의 사실 근거성 (Faithfulness)** | 답변이 제공된 컨텍스트(문서, 코드 등)에 기반하여 사실적으로 정확한지를 평가합니다. (Hallucination 방지) |
| | **3. 검색된 컨텍스트 관련성 (Context Relevancy)** | RAG 시스템이 질문에 답변하기 위해 검색한 컨텍스트가 질문과 얼마나 관련이 있는지를 평가합니다. |
| | **4. 답변의 정답 정확성 (Answer Correctness)** | 정답이 명확한 질문에 대해, 답변이 사실과 일치하는지를 평가합니다. |
| **품질** | **5. 답변의 유용성 (Helpfulness)** | 답변이 사용자의 문제를 해결하거나 정보를 제공하는 데 실질적으로 얼마나 도움이 되는지를 평가합니다. |
| | **6. 답변의 논리적 일관성 (Coherence)** | 답변의 문장 구조가 논리적이고, 문법적으로 자연스러우며, 이해하기 쉬운지를 평가합니다. |
| | **7. 답변의 간결성 (Conciseness)** | 답변이 불필요한 정보 없이 핵심 내용을 간결하게 전달하는지를 평가합니다. |
| **일관성**| **8. 다중 대화 일관성 (Multi-turn Coherence)**| 여러 차례의 대화에서 이전의 맥락을 기억하고 일관된 답변을 유지하는지를 평가합니다. |
| **안전성** | **9. 개인정보 탐지 (PII Detection)** | 답변에 주민등록번호, 카드번호 등 민감한 개인정보가 포함되어 있는지 여부를 검사합니다. |
| | **10. 유해성 발언 탐지 (Toxicity)** | 답변에 혐오, 공격, 비속어 등 부적절하거나 유해한 내용이 포함되어 있는지 여부를 검사합니다. |
| **사용자 요구사항**| **11. 사용자 정의 기준 충족 (Custom Criteria)** | "특정 단어 포함" 등 각 평가 케이스별로 사용자가 정의한 특정 성공 기준을 충족하는지 여부를 검사합니다. |

## 4. 사용 방법

### 평가 데이터셋 준비

Excel 또는 텍스트 편집기를 사용하여 "질문(prompt)"과 "성공 기준(criteria)"을 포함하는 테이블을 작성한 후, CSV (UTF-8) 형식으로 저장합니다. 
평가의 신뢰도를 높이기 위해, 기능 검증, 안전성, 다중 대화 등 다양한 유형을 포괄하는 10건 이상의 평가 케이스를 준비할 것을 권장합니다.

### 평가 실행

1.  Jenkins에서 "DSCORE-Agent-Eval" 작업을 선택합니다.
2.  평가 대상 AI의 접속 주소와 호출 방식(API 또는 Web)을 파라미터를 통해 지정합니다.
3.  준비된 평가 데이터셋(CSV 파일)을 업로드합니다.
4.  작업을 실행합니다.

### 결과 확인

실행이 완료되면 Jenkins 빌드 페이지의 'AI Eval Summary' 탭에서 시각적 요약 리포트를 확인하거나, 빌드 산출물(Artifacts)에서 상세 HTML 리포트 파일을 다운로드할 수 있습니다. 
평가 대시보드(Langfuse)가 연동된 경우, 해당 대시보드에서 누적된 평가 결과의 추이를 시계열 그래프로 확인할 수 있습니다.

## 5. 주요 기대 효과 및 활용 시나리오

본 파이프라인은 AI 에이전트의 품질을 객관적으로 관리하기 위한 다양한 활용 시나리오를 제공합니다.

-   **AI 모델 성능 비교 분석 (A/B 테스트):** 신규 AI 모델을 도입하거나 기존 모델을 업데이트하기 전, 동일한 평가 데이터셋을 사용하여 두 모델의 성능을 각 지표별로 정량적으로 비교하고, 데이터에 기반하여 도입 여부를 결정할 수 있습니다.
-   **챗봇 서비스 품질 정기 검증:** 고객 응대 챗봇 등을 운영하는 경우, 주요 고객 질의로 구성된 데이터셋을 통해 주기적으로 품질을 검증하고, 특정 유형의 질문에 대한 답변 품질 저하 여부를 모니터링할 수 있습니다.
-   **AI 안전성 및 보안성 검증 자동화:** 개인정보나 금칙어 관련 평가 케이스를 포함하여, AI가 민감 정보를 노출하거나 유해한 발언을 생성하지 않는지를 자동으로 검증하는 안전성 회귀 테스트를 구축할 수 있습니다.
-   **정량적 성과 보고:** AI 서비스의 품질 현황 보고 시, "답변 정확도 92%, 유해 발언 탐지 0건" 등 평가 리포트에서 도출된 객관적인 수치를 핵심 성과 지표(KPI)로 활용할 수 있습니다.

## 6. 참고 사항

-   응답 품질 평가는 사내 로컬 서버에서 구동되는 LLM이 수행하므로, 평가 데이터와 AI의 응답 내용이 외부로 유출되지 않습니다.
-   평가 대시보드(Langfuse) 연동은 선택 사항이며, 설정하지 않아도 Jenkins 내에서 평가 리포트를 확인할 수 있습니다.
-   평가 데이터셋 파일은 반드시 UTF-8 인코딩으로 저장해야 합니다.
