# 파이프라인 소개: 자율 E2E 테스트 (Job #7)

## 1. 개요

Zero-Touch QA는 **자연어 요구사항만 입력하면 AI가 E2E 테스트를 자동으로 생성하고 실행하는** 자율 주행 QA 시스템이다.

기존 E2E 테스트의 최대 문제는 **깨지기 쉽다(Flaky)**는 것이다. 버튼 텍스트가 "로그인"에서 "Login"으로 바뀌면 `page.click("text=로그인")` 테스트가 즉시 실패한다. Zero-Touch QA는 이 문제를 **7단계 시맨틱 탐색**과 **하이브리드 자가 치유**로 해결한다.

---

## 2. 핵심 설계 원칙

### 두뇌(Brain)와 근육(Muscle)의 분리

| 계층 | 담당 | 구현 |
| --- | --- | --- |
| **지능 계층 (Dify Brain)** | 무엇을 해야 할지 판단 | Dify Chatflow의 Planner/Vision/Healer LLM 노드 |
| **실행 계층 (Python Executor)** | 실제 브라우저를 조작 | mac_local_executor.py + Playwright |

지능 계층은 프롬프트 수정만으로 업그레이드할 수 있고, 실행 계층은 자체 탐색/치유 로직으로 강건한 생존력을 가진다.

### 9대 표준 DSL

AI의 환각(Hallucination)을 방지하기 위해, 두 계층은 오직 9개의 표준 액션으로만 소통한다.

| 액션 | 하는 일 | 예시 |
| --- | --- | --- |
| `navigate` | URL로 이동 | 네이버 메인 페이지로 이동 |
| `click` | 요소 클릭 | 로그인 버튼 클릭 |
| `fill` | 텍스트 입력 | 검색창에 "DSCORE" 입력 |
| `press` | 키보드 키 입력 | Enter 키 누르기 |
| `select` | 드롭다운 선택 | 국가 목록에서 "한국" 선택 |
| `check` | 체크박스 토글 | 약관 동의 체크 |
| `hover` | 마우스 호버 | 메뉴 항목에 마우스 올리기 |
| `wait` | 지정 시간 대기 | 페이지 로딩 2초 대기 |
| `verify` | 텍스트/요소 확인 | 결과 영역에 "성공" 텍스트가 있는지 확인 |

## 3. 3가지 진입 경로 (3-Flow)

사용자는 코드를 작성할 필요 없이, 상황에 맞는 방식으로 테스트를 시작한다.

### Flow 1: 문서 업로드 (Doc-to-Test)

기획서나 요구사항 정의서(PDF/Word)를 업로드하면, AI가 문서를 분석하여 테스트 시나리오를 자동 생성한다.

**적합한 경우:** 기획서 기반으로 대량의 테스트 시나리오를 한 번에 구축할 때

### Flow 2: 자연어 입력 (Chat-to-Test)

Dify 채팅창이나 Jenkins 파라미터에 자연어로 지시하면, AI가 즉시 테스트 시나리오로 변환한다.

**적합한 경우:** 새로운 기능을 빠르게 검증하거나 디버깅할 때

**입력 예시:** "네이버에 접속해서 검색창에 DSCORE를 입력하고 엔터를 눌러줘"

### Flow 3: 브라우저 녹화 (Record-to-Test)

브라우저를 평소처럼 조작하면, 시스템이 클릭/입력 이벤트를 캡처하여 테스트 시나리오로 변환한다.

**적합한 경우:** 복잡한 UI 조작을 직접 시연하여 테스트로 자산화할 때

## 4. 7단계 시맨틱 탐색 (LocatorResolver)

일반 테스트 도구는 CSS 셀렉터(`#login-btn`)에 의존하여 UI가 변경되면 실패한다. Zero-Touch QA는 7단계에 걸쳐 요소를 탐색하며, 상위 단계일수록 UI 변경에 강하다.

| 단계 | 탐색 방법 | 예시 | UI 변경 내성 |
| --- | --- | --- | --- |
| 1 | 접근성 역할 + 이름 | `role=button, name=로그인` | 가장 강함 |
| 2 | 텍스트 | `text=로그인` | 강함 |
| 3 | 라벨 | `label=이메일` | 강함 |
| 4 | 플레이스홀더 | `placeholder=검색어 입력` | 보통 |
| 5 | 테스트 ID | `testid=login-button` | 보통 |
| 6 | CSS/XPath | `#login-btn` | 약함 |
| 7 | 존재 검증 | 위 6단계 모두 실패 시 → 치유 전환 | - |

---

## 5. 하이브리드 자가 치유 (Self-Healing)

7단계 탐색이 모두 실패하면, 자가 치유가 작동한다.

| 단계 | 방식 | 비용 | 속도 |
| --- | --- | --- | --- |
| 1단계 | **Candidate Search** — 현재 페이지의 DOM을 스캔하여 유사한 요소를 찾음 (유사도 80% 이상) | 무료 (로컬) | ~10ms |
| 2단계 | **Dify Healer LLM** — DOM 스냅샷과 에러를 LLM에 전송하여 대체 셀렉터를 제안받음 | LLM 호출 | ~3초 |

로컬 치유를 먼저 시도하고, 실패할 때만 LLM을 호출하여 비용과 속도를 최적화한다.
