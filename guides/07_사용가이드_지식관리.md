# 사용 가이드: 지식 관리 파이프라인 (Job #1~#4, #6)

## 사전 준비

- Jenkins에 로그인할 수 있어야 한다 (`http://localhost:8080`)
- Dify에 Knowledge Base(Dataset)가 생성되어 있어야 한다
- Jenkins Credentials에 `dify-knowledge-key`, `dify-dataset-id` 등이 등록되어 있어야 한다

---

## Job #1: 문서 업로드 (DSCORE-Knowledge-Sync)

### 실행 전 준비

AI에 학습시킬 문서를 호스트 머신의 다음 경로에 복사한다:

```
<PROJECT_ROOT>/data/knowledges/docs/org/
```

지원 형식: PDF, DOCX, XLSX, TXT, PPTX

### 실행 방법

1. Jenkins에서 `DSCORE-Knowledge-Sync` Job을 클릭한다.
2. 좌측 메뉴에서 `지금 빌드(Build Now)`를 클릭한다.
3. 별도의 파라미터 입력은 필요 없다.

### 실행 결과 확인

1. 빌드 완료 후 `Console Output`을 클릭하여 로그를 확인한다.
2. 변환된 파일은 다음 경로에서 확인할 수 있다:
   ```
   <PROJECT_ROOT>/data/knowledges/docs/result/
   ```
3. Dify 웹 UI에 접속하여 Knowledge Base에 문서가 등록되었는지 확인한다:
   - Dify 로그인 → 좌측 메뉴 `Knowledge` → 해당 Dataset 클릭 → 문서 목록 확인

### 재실행 시 참고사항

- 동일 파일을 다시 업로드하면 Dify에서 중복 문서가 생성될 수 있다.
- 문서를 갱신하려면 Dify에서 기존 문서를 삭제한 후 재실행하는 것을 권장한다.

---

## Job #2: Q&A 업로드 (DSCORE-Knowledge-Sync-QA)

### 실행 전 준비

Job #1과 동일한 원본 문서 폴더를 사용한다. 별도의 파일 준비는 필요 없다.

### 실행 방법

1. Jenkins에서 `DSCORE-Knowledge-Sync-QA` Job을 클릭한다.
2. `지금 빌드(Build Now)`를 클릭한다.

### 실행 결과 확인

1. Dify 웹 UI에서 **Q&A형 Dataset**에 질문-답변 쌍이 등록되었는지 확인한다.
2. Q&A형과 문서형은 **별도의 Dataset**이다. Dify에서 각각의 Dataset을 확인해야 한다.

---

## Job #3: 이미지 분석 (DSCORE-Knowledge-Sync-Vision)

### 실행 전 준비

1. 분석할 이미지 파일을 `<PROJECT_ROOT>/data/knowledges/docs/org/`에 복사한다.
2. **Ollama가 호스트 머신에서 실행 중이어야 한다.**
   ```bash
   # Ollama 실행 확인
   ollama list
   # llama3.2-vision 모델이 설치되어 있어야 한다
   ollama pull llama3.2-vision
   ```

### 실행 방법

1. Jenkins에서 `DSCORE-Knowledge-Sync-Vision` Job을 클릭한다.
2. `지금 빌드(Build Now)`를 클릭한다.

### 실행 결과 확인

1. Console Output에서 각 이미지에 대한 Vision AI 분석 결과를 확인한다.
2. 분석된 설명 텍스트가 Dify Knowledge Base에 업로드된다.

---

## Job #4: 코드 컨텍스트 (DSCORE-Code-Knowledge-Sync)

### 실행 전 준비

GitLab에 분석 대상 프로젝트가 등록되어 있어야 한다.

### 실행 방법

1. Jenkins에서 `DSCORE-Code-Knowledge-Sync` Job을 클릭한다.
2. `Build with Parameters`를 클릭한다.
3. 파라미터를 입력한다:

| 파라미터 | 입력 예시 | 설명 |
| --- | --- | --- |
| `REPO_URL` | `http://gitlab:8929/root/my-project.git` | 분석 대상 Git 저장소 URL |

4. `빌드`를 클릭한다.

### 실행 결과 확인

1. Console Output에서 프로젝트 구조 분석 결과를 확인한다.
2. 생성된 `context.md` 파일이 Dify Knowledge Base에 업로드된다.
3. Dify 챗봇에서 "이 프로젝트의 기술 스택은?" 같은 질문을 던져 학습 결과를 확인할 수 있다.

---

## Job #6: 웹 크롤링 (DSCORE-Web-Knowledge-Sync)

### 실행 전 준비

별도의 사전 준비는 필요 없다. 크롤링 대상 URL만 결정하면 된다.

### 실행 방법

1. Jenkins에서 `DSCORE-Web-Knowledge-Sync` Job을 클릭한다.
2. `Build with Parameters`를 클릭한다.
3. 파라미터를 입력한다:

| 파라미터 | 입력 예시 | 설명 |
| --- | --- | --- |
| `ROOT_URL` | `https://docs.example.com` | 크롤링 시작 URL |

4. `빌드`를 클릭한다.

### 실행 결과 확인

1. Console Output에서 크롤링된 페이지 수와 추출된 텍스트를 확인한다.
2. 추출된 내용이 `web_*.md` 파일로 저장되고 Dify에 업로드된다.

### 주의사항

- 외부 웹사이트 크롤링 시 해당 사이트의 `robots.txt` 정책을 준수해야 한다.
- 대량 크롤링은 대상 서버에 부하를 줄 수 있으므로 적절한 범위로 제한한다.

---

## 전체 지식 베이스 구축 순서 (처음 사용 시)

처음으로 전체 지식 베이스를 구축할 때의 권장 순서:

```
1. 문서를 <PROJECT_ROOT>/data/knowledges/docs/org/ 에 복사
2. Job #1 실행 (문서 업로드)
3. Job #2 실행 (Q&A 업로드)
4. Job #3 실행 (이미지 분석) — 이미지가 있는 경우
5. Job #4 실행 (코드 컨텍스트) — Git 저장소가 있는 경우
6. Job #6 실행 (웹 크롤링) — 참고할 웹 문서가 있는 경우
7. Dify 챗봇에서 학습 결과 확인
```

이후에는 변경된 항목에 해당하는 Job만 재실행하면 된다.
