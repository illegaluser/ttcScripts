# 파이프라인 소개: 지식 관리 자동화 (Job #1~#4, #6)

## 1. 개요

지식 관리 파이프라인은 프로젝트의 문서, 코드, 이미지, 웹 콘텐츠를 **Dify Knowledge Base(AI 지식 저장소)**에 자동으로 업로드하여, LLM이 프로젝트 맥락을 이해하고 정확한 답변을 제공할 수 있도록 만드는 자동화 체계이다.

이 파이프라인이 실행되면, Dify에 연결된 AI 챗봇이나 워크플로우에서 "우리 프로젝트의 로그인 로직이 어떻게 구현되어 있어?"와 같은 질문에 실제 코드와 문서를 기반으로 답변할 수 있게 된다.

---

## 2. 파이프라인 목록

### Job #1: DSCORE-Knowledge-Sync (문서형 업로드)

**하는 일:** PDF, DOCX, XLSX, TXT, PPTX 파일을 Markdown 텍스트로 변환한 후 Dify Knowledge Base에 업로드한다.

**변환 과정:**
- PDF → PyMuPDF로 텍스트 추출 → Markdown
- DOCX → python-docx로 문단 추출 → Markdown
- XLSX → pandas로 시트/테이블 추출 → Markdown 표
- PPTX → LibreOffice로 PDF 변환 → PyMuPDF로 텍스트 추출
- TXT → 그대로 Markdown으로 감싸기

**결과:** Dify의 문서형(text_model) Dataset에 문서가 등록되어, AI가 문서 내용을 검색하고 참조할 수 있게 된다.

---

### Job #2: DSCORE-Knowledge-Sync-QA (Q&A형 업로드)

**하는 일:** Job #1과 동일한 원본 문서를 **Q&A 형태**로 Dify에 업로드한다.

**문서형과 Q&A형의 차이:**
- **문서형(Job #1):** 원문 텍스트를 그대로 청킹(분할)하여 저장. "이 문서에 뭐라고 적혀 있어?" 류의 질문에 적합.
- **Q&A형(Job #2):** Dify가 문서에서 질문-답변 쌍을 자동 생성하여 저장. "~는 어떻게 하면 돼?" 류의 질문에 더 정확한 답변 제공.

**결과:** Dify의 Q&A형(qa_model) Dataset에 질문-답변 쌍이 등록된다.

---

### Job #3: DSCORE-Knowledge-Sync-Vision (이미지 분석)

**하는 일:** 이미지 파일(PNG, JPG 등)을 **Ollama Vision 모델**(llama3.2-vision)로 분석하여 설명 텍스트를 생성하고, 그 설명을 Dify에 업로드한다.

**동작 예시:**
1. `architecture_diagram.png` 파일을 Vision AI에 전달
2. AI가 "이 다이어그램은 3-tier 웹 애플리케이션 구조를 보여주며, 프론트엔드(React), 백엔드(Spring Boot), 데이터베이스(PostgreSQL)로 구성되어 있다..."라고 설명 생성
3. 생성된 설명 텍스트를 Dify Knowledge Base에 업로드

**결과:** 이미지에 담긴 정보까지 AI가 텍스트로 이해하고 답변에 활용할 수 있게 된다.

---

### Job #4: DSCORE-Code-Knowledge-Sync (코드 컨텍스트)

**하는 일:** Git 저장소를 클론하여 프로젝트 구조와 핵심 파일을 분석하고, 이를 하나의 컨텍스트 문서(`context.md`)로 만들어 Dify에 업로드한다.

**분석 대상:**
- 프로젝트 디렉터리 트리 구조
- README.md, package.json, pom.xml 등 프로젝트 설정 파일
- 주요 소스 코드 파일의 핵심 내용

**결과:** AI가 "이 프로젝트는 어떤 기술 스택을 사용하고 있어?", "src/auth/ 폴더의 역할이 뭐야?" 같은 질문에 답변할 수 있게 된다.

---

### Job #6: DSCORE-Web-Knowledge-Sync (웹 콘텐츠 수집)

**하는 일:** 지정된 웹사이트를 **Crawl4AI**로 크롤링하여 텍스트를 추출하고, 정제한 후 Dify에 업로드한다.

**정제 과정:**
- HTML에서 본문 텍스트만 추출 (메뉴, 푸터, 광고 제거)
- URL, 이메일 주소 등 노이즈 제거
- 깨끗한 Markdown 형태로 저장

**활용 예시:**
- 사내 위키/기술 블로그를 AI 지식으로 흡수
- 외부 기술 문서(예: React 공식 문서)를 프로젝트 컨텍스트에 추가

**결과:** 웹에 산재한 정보를 AI가 통합적으로 검색하고 답변에 활용할 수 있게 된다.

---

## 3. 실행 순서 권장

```
Job #1 (문서 업로드)  →  Job #2 (Q&A 업로드)  →  Job #3 (이미지 분석)
                                                       ↓
Job #4 (코드 컨텍스트)  →  Job #6 (웹 크롤링)  →  Dify Knowledge Base 완성
```

Job #1~#4, #6은 독립적으로 실행 가능하지만, 전체 지식 베이스를 구축하려면 위 순서를 권장한다. 한 번 실행한 후에는 문서가 갱신될 때마다 해당 Job만 재실행하면 된다.
