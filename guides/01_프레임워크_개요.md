# DSCORE-TTC 프레임워크 개요

## 한 줄 요약

DSCORE-TTC는 **AI가 코드 품질을 진단하고, 테스트를 스스로 작성하며, 외부 AI의 성능을 자동으로 채점하는** 온프레미스 DevOps 통합 플랫폼이다.

---

## 1. 왜 만들었는가

소프트웨어 개발 현장에서 반복적으로 발생하는 세 가지 병목이 있다.

**병목 1: 지식이 사람 머릿속에만 있다**

프로젝트 문서, 코드 컨텍스트, 기술 레퍼런스가 흩어져 있어 신규 인력이 업무에 투입되기까지 시간이 오래 걸린다. AI에게 "우리 프로젝트에서 로그인 로직이 어떻게 구현되어 있어?"라고 물어볼 수 없다.

**병목 2: 코드 리뷰와 이슈 등록이 수동이다**

SonarQube가 코드 품질 문제를 발견해도, 그 결과를 해석하고 GitLab Issue로 정리하는 건 사람 몫이다. 수십 개의 이슈를 일일이 분류하고 설명을 작성하는 데 반나절이 소요된다.

**병목 3: UI 테스트가 깨지기 쉽다**

Playwright나 Selenium으로 E2E 테스트를 작성해도, 버튼 텍스트 하나 바뀌면 테스트가 실패한다. 테스트 유지보수 비용이 개발 비용을 초과하는 경우가 발생한다.

DSCORE-TTC는 이 세 가지 병목을 **AI + 자동화 파이프라인**으로 해결한다.

---

## 2. 무엇을 하는가

### 5단계 자동화 체계

| 단계 | 이름 | 하는 일 | 관련 Job |
| --- | --- | --- | --- |
| Phase 1 | DevOps 인프라 | Jenkins, SonarQube, GitLab, Dify, Langfuse를 Docker로 구동 | (인프라) |
| Phase 2 | 지식 관리 | 문서, 코드, 이미지, 웹 콘텐츠를 AI가 학습할 수 있도록 Dify에 자동 업로드 | Job #1~#4, #6 |
| Phase 3 | 품질 분석 | SonarQube 분석 결과를 LLM이 진단하고 GitLab Issue를 자동 생성 | Job #5 |
| Phase 4 | 자율 E2E 테스트 | 자연어 요구사항만으로 브라우저 E2E 테스트를 자동 생성 및 실행 | Job #7 |
| Phase 5 | AI 에이전트 평가 | 외부 AI(챗봇, API)를 11대 지표로 자동 채점 | Job #8 |

### 8개 Jenkins 파이프라인

| Job | 이름 | 한 줄 설명 |
| --- | --- | --- |
| #1 | DSCORE-Knowledge-Sync | 문서(PDF/Word/Excel)를 AI 지식 베이스에 업로드 |
| #2 | DSCORE-Knowledge-Sync-QA | 문서를 Q&A 형태로 변환하여 업로드 |
| #3 | DSCORE-Knowledge-Sync-Vision | 이미지를 Vision AI로 분석하여 설명 텍스트 업로드 |
| #4 | DSCORE-Code-Knowledge-Sync | Git 저장소 구조와 핵심 코드를 분석하여 업로드 |
| #5 | DSCORE-Quality-Issue-Workflow | 정적 분석 → LLM 진단 → GitLab Issue 자동 등록 |
| #6 | DSCORE-Web-Knowledge-Sync | 웹사이트를 크롤링하여 텍스트 지식 업로드 |
| #7 | DSCORE-ZeroTouch-QA | 자연어 기반 자율 E2E 테스트 (Dify Brain + Self-Healing) |
| #8 | DSCORE-Agent-Eval | 외부 AI 에이전트 11대 지표 자동 평가 |

---

## 3. 누가 사용하는가

| 역할 | 주로 사용하는 기능 |
| --- | --- |
| **개발자** | Job #1~#4로 프로젝트 지식을 AI에 학습시키고, Job #5로 코드 품질 이슈를 자동 관리 |
| **QA 엔지니어** | Job #7로 자연어 요구사항만 입력하면 E2E 테스트가 자동 생성 및 실행됨 |
| **AI/ML 엔지니어** | Job #8로 개발한 AI 모델/챗봇의 품질을 11대 지표로 자동 측정 |
| **기획자/PO** | Job #7에 기획서(PDF)를 업로드하면 테스트 시나리오가 자동 생성됨 |
| **팀 리드** | Langfuse 대시보드에서 AI 품질 추이를 실시간 모니터링 |

---

## 4. 기술 스택 요약

| 구분 | 기술 | 역할 |
| --- | --- | --- |
| CI/CD | Jenkins | 파이프라인 실행 및 스케줄링 |
| 코드 품질 | SonarQube | 정적 코드 분석 (버그, 취약점, 코드 스멜) |
| 소스 관리 | GitLab | Git 저장소, 이슈 트래커 |
| AI 플랫폼 | Dify | LLM 워크플로우 설계, 지식 베이스(RAG), Chatflow |
| AI 관측성 | Langfuse | LLM 호출 추적, 평가 점수 대시보드 |
| 로컬 LLM | Ollama | 온프레미스 LLM 실행 (qwen3-coder, llama3.2-vision 등) |
| E2E 테스트 | Playwright | 브라우저 자동화 엔진 |
| AI 평가 | DeepEval + Promptfoo | LLM 응답 심층 채점 + 보안 검사 |
| 인프라 | Docker Compose | 전체 서비스 컨테이너화 및 네트워크 구성 |

## 5. 접속 정보

시스템이 구축된 후 브라우저에서 접속할 수 있는 주소는 다음과 같다.

| 서비스 | URL | 용도 |
| --- | --- | --- |
| Jenkins | http://localhost:8080 | 파이프라인 실행 및 관리 |
| SonarQube | http://localhost:9000 | 정적 분석 결과 확인 |
| GitLab | http://localhost:8929 | 소스 코드 및 이슈 관리 |
| Langfuse | http://localhost:3000 | AI 평가 결과 대시보드 |
| Dify | http://localhost (기본) | AI 워크플로우 및 지식 베이스 관리 |
